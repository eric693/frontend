核心概念 01

選擇器與優先權

CSS 最讓初學者困惑的問題之一：「我明明寫了這個樣式，為什麼沒有生效？」答案通常在於優先權（Specificity）。

01
CSS 引入方式
Inline / Internal / External

CSS 有三種引入方式，優先權由高到低：行內樣式（直接寫在 style 屬性）、內部樣式表（<style> 標籤）、外部樣式表（連結 .css 檔案）。

雖然行內樣式優先權最高，但這不代表要大量使用它。外部樣式表是最佳實踐——它讓 HTML 專注結構、CSS 專注樣式，多個頁面可共用同一份樣式表，也便於維護和覆寫。

避免 不要大量使用行內樣式（style="color:red"）或 !important。這樣做雖然能暫時解決問題，但會讓你的 CSS 變得難以維護，並產生難以預測的優先權衝突。
01
特異性計算
Specificity Score

每個 CSS 選擇器都有一個特異性分數，由三個數字組成：ID 數量、類別/屬性/偽類數量、元素/偽元素數量。分數越高的規則越優先。

當兩個選擇器特異性相同時，後寫的規則覆蓋先寫的。這就是「層疊」（Cascade）的意義——CSS 的 C 就是 Cascading。

CSS — 選擇器類型與特異性計算
/* ── 選擇器類型 ── */

*              { color: gray; }    /* 通用選擇器 — 特異性 0,0,0 */
p              { color: black; }   /* 元素選擇器 — 0,0,1 */
.card          { color: blue; }    /* 類別選擇器 — 0,1,0 */
#header        { color: red; }     /* ID 選擇器 — 1,0,0 */

/* ── 組合選擇器 ── */

nav a          { color: teal; }    /* 後代選擇器 — 0,0,2 */
nav > a        { color: teal; }    /* 直接子元素 — 0,0,2 */
h2 + p        { color: teal; }    /* 相鄰兄弟 — 0,0,2 */

/* ── 偽類與屬性選擇器 ── */

a:hover        { color: orange; }  /* 偽類 — 0,1,1 */
input[type="text"] { border: 1px solid #ccc; } /* 屬性 — 0,1,1 */
li:nth-child(2n)  { background: #f5f5f5; } /* 0,1,1 */

/* ── 特異性計算範例 ── */

.nav a:hover   { color: red; }    /* 0,1,1 + 0,1,0 = 0,2,1 */
#header .logo  { color: blue; }   /* 1,0,0 + 0,1,0 = 1,1,0 → 勝出 */

el el後代選擇器選取特定容器內的所有後代

::before / ::after偽元素在元素前後插入裝飾性內容
核心概念 02

盒模型

CSS 中每個元素都是一個盒子，由四個層次組成。理解盒模型是掌握版面的基礎，也是解決「為什麼我的版面跑掉了」的關鍵。

02
四層結構
content / padding / border / margin

content 是元素實際顯示內容的區域，width 和 height 預設只控制這個區域。

padding 是 content 與 border 之間的內間距，背景色會延伸到 padding 區域。

border 是邊框，可設定寬度、樣式、顏色。

margin 是元素與相鄰元素之間的外間距，背景色不會延伸到這裡。注意：垂直方向的 margin 會發生「外邊距折疊」（margin collapse），兩個相鄰的 margin 取較大值，而不是相加。

02
box-sizing: border-box
最重要的 CSS Reset

預設情況下，width: 200px 只是 content 的寬度。加上 padding: 20px 和 border: 2px，元素的實際佔用寬度是 244px（200 + 20×2 + 2×2），這經常讓初學者困惑。

設定 box-sizing: border-box 後，width: 200px 代表元素的總寬度包含 padding 和 border，讓尺寸計算更直覺。幾乎所有現代專案都會在全域設定這個屬性。

✕ content-box（預設，計算複雜）
.box {
  width: 200px;
  padding: 20px;
  border: 2px solid;
  /* 實際寬度：244px */
  /* 200 + 20*2 + 2*2 */
}
✓ border-box（直覺，推薦）
.box {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 2px solid;
  /* 實際寬度：200px */
}
CSS — 全域 box-sizing 與常用盒模型屬性
/* ── 全域重設（幾乎每個專案都需要）── */
*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

/* ── padding 簡寫 ── */
.card {
  padding: 16px;           /* 四邊相同 */
  padding: 16px 24px;     /* 上下 16px，左右 24px */
  padding: 8px 16px 24px 12px; /* 上右下左（順時針）*/
}

/* ── margin 自動居中 ── */
.container {
  max-width: 1200px;
  margin: 0 auto;  /* 上下0，左右自動 = 水平置中 */
  padding: 0 1.5rem;
}

/* ── border 語法 ── */
.button {
  border: 1px solid #e2e8f0;
  border-radius: 6px;    /* 圓角 */
  border-radius: 50%;    /* 圓形（需要正方形元素）*/
}

/* ── display 屬性 ── */
span   { display: inline; }        /* 行內：不換行，無法設寬高 */
div    { display: block; }         /* 區塊：自動換行，可設寬高 */
.icon { display: inline-block; }  /* 兩者兼顧 */
核心概念 03

Flexbox

Flexbox 是一維排版工具，專門解決一個方向（橫向或縱向）的排列問題。它是現代 CSS 最常用的佈局方式之一，掌握它能解決大多數排版需求。

03
Flex Container vs Flex Items
容器屬性 vs 子元素屬性

Flexbox 分為兩個角色：Flex 容器（設定 display: flex 的父元素）和 Flex 子元素（容器的直接子元素）。容器屬性控制整體排列方式，子元素屬性控制自身的大小和對齊。

主軸（Main Axis）是排列方向，預設為水平（row）。交叉軸（Cross Axis）與主軸垂直。justify-content 控制主軸對齊，align-items 控制交叉軸對齊。記住這組概念，大部分 Flexbox 問題就迎刃而解。

常見需求：垂直水平置中 在父元素設定 display: flex; justify-content: center; align-items: center;，子元素就能完美置中於容器內。這個技巧在現代 CSS 中幾乎取代了過去所有置中的 hack。

CSS — Flexbox 完整參考
/* ── 容器屬性 ── */
.flex-container {
  display: flex;

  /* 排列方向 */
  flex-direction: row;         /* 預設：水平由左至右 */
  flex-direction: column;      /* 垂直由上至下 */
  flex-direction: row-reverse; /* 水平由右至左 */

  /* 主軸對齊（水平，當 direction=row）*/
  justify-content: flex-start;    /* 靠左 */
  justify-content: center;        /* 置中 */
  justify-content: flex-end;      /* 靠右 */
  justify-content: space-between; /* 兩端對齊，中間等間 */
  justify-content: space-evenly;  /* 所有間距相等 */

  /* 交叉軸對齊（垂直，當 direction=row）*/
  align-items: center;   /* 垂直置中 */
  align-items: stretch;  /* 拉伸填滿高度（預設）*/
  align-items: flex-start; /* 對齊頂部 */

  /* 換行 */
  flex-wrap: wrap;     /* 超出時自動換行 */
  flex-wrap: nowrap;   /* 預設：不換行 */
  gap: 16px;            /* 子元素間距（現代瀏覽器均支援）*/
}

/* ── 子元素屬性 ── */
.flex-item {
  flex-grow: 1;     /* 佔用剩餘空間的比例（0=不增長）*/
  flex-shrink: 0;   /* 空間不足時是否縮小（0=不縮小）*/
  flex-basis: 200px;/* 初始寬度 */
  flex: 1 0 200px;  /* 簡寫：grow shrink basis */
  flex: 1;           /* 等比例分配剩餘空間 */
  align-self: flex-end; /* 覆蓋容器的 align-items */
  order: 2;          /* 視覺排列順序（不改變 DOM 順序）*/
}
屬性作用常用值
justify-content主軸對齊方式flex-start / center / space-between / space-evenly
align-items交叉軸對齊（單行）stretch / center / flex-start / baseline
flex-wrap是否允許換行nowrap / wrap / wrap-reverse
gap子元素間距像素值，如 gap: 16px 或 gap: 8px 16px
flex-direction主軸方向row / column / row-reverse / column-reverse
flex子元素大小（簡寫）flex: 1 等比例，flex: 0 0 auto 固定大小
align-self個別子元素的交叉軸對齊同 align-items 的值
核心概念 04

CSS Grid

Grid 是二維排版工具，同時控制欄（column）和列（row）。適合用在整個頁面的大版面架構，或需要精確對齊的卡片系統。

04
Grid vs Flexbox：如何選擇？
一維 vs 二維

這兩者不是競爭關係，而是互補：Flexbox 適合一維排列——一排按鈕、一列導覽項目、卡片組件的內部佈局。Grid 適合二維版面——整個頁面的欄位架構、需要橫跨多欄或多列的版面。

實際上，兩者常常一起使用：Grid 定義頁面的大區塊，Flexbox 處理各個區塊內部的排列。

CSS — Grid 完整參考
/* ── Grid 容器 ── */
.grid-container {
  display: grid;

  /* 定義欄位 */
  grid-template-columns: 200px 1fr 1fr;    /* 固定+彈性 */
  grid-template-columns: repeat(3, 1fr);   /* 三等份 */
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr)); /* 響應式卡片 */

  /* 定義列高 */
  grid-template-rows: auto 1fr auto;   /* header / main / footer */

  /* 間距 */
  gap: 24px;             /* 欄列間距相同 */
  gap: 16px 24px;        /* 列間距 16px，欄間距 24px */

  /* 對齊 */
  justify-items: start;   /* 格內橫向對齊 */
  align-items: center;   /* 格內縱向對齊 */
}

/* ── Grid 子元素：跨欄/列 ── */
.header {
  grid-column: 1 / -1;    /* 從第1欄到最後一欄（橫跨全寬）*/
  grid-column: span 2;    /* 佔 2 欄 */
}

.sidebar {
  grid-row: 2 / 4;         /* 跨越第 2 到第 4 列 */
}

/* ── 實用版面範例 ── */
.page-layout {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 60px 1fr 60px;
  min-height: 100vh;
}
.page-layout header  { grid-column: 1 / -1; }
.page-layout footer  { grid-column: 1 / -1; }

/* ── 響應式卡片 grid（最常用技巧）── */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 1.5rem;
  /* 自動填充，每張卡片最小 280px，空間夠就自動排多列 */
}
屬性作用範例
grid-template-columns定義欄的數量和寬度repeat(3, 1fr) / 200px 1fr
grid-template-rows定義列的高度auto 1fr auto
gap格線間距16px 或 8px 16px
grid-column子元素佔用的欄範圍1 / -1（全寬）/ span 2
grid-row子元素佔用的列範圍2 / 4 / span 3
auto-fill vs auto-fit自動填充模式repeat(auto-fill, minmax(200px, 1fr))
fr 單位彈性比例單位1fr 代表剩餘空間的一份
最實用技巧：auto-fill + minmax grid-template-columns: repeat(auto-fill, minmax(280px, 1fr)) 是製作響應式卡片列表的神奇公式。它告訉瀏覽器：「盡量多排幾欄，每欄最小 280px，剩餘空間平均分配。」寬度夠寬時自動 3 欄，縮窄後自動變 2 欄，再窄變 1 欄，完全不需要媒體查詢。
核心概念 05

媒體查詢與 RWD

響應式設計（Responsive Web Design）讓同一份 HTML/CSS 在手機、平板、桌面都能良好呈現。媒體查詢是實現 RWD 的核心工具。

05
Mobile-First 設計思維
從小螢幕開始設計

有兩種設計策略：Mobile-First 先寫手機版樣式，再用 min-width 媒體查詢漸進增強到大螢幕；Desktop-First 則相反，先寫桌面版再用 max-width 縮減。

現代開發推薦 Mobile-First，原因是：手機版通常是最簡單的版面，基礎樣式應該是最通用的，而桌面版的複雜度是在手機版基礎上「增加」的，這樣的 CSS 結構更乾淨，效能也更好（手機不會載入不需要的桌面樣式）。

HEAD 中的 viewport meta 標籤 記得在 <head> 中加上 <meta name="viewport" content="width=device-width, initial-scale=1.0">，否則手機會把桌面版頁面縮小顯示，媒體查詢也不會正常作用。
CSS — 媒體查詢語法與 Mobile-First 策略
/* ── 媒體查詢基本語法 ── */
@media (min-width: 768px) {
  /* 只在 768px 以上才套用的樣式 */
}

@media (max-width: 767px) {
  /* 只在 767px 以下才套用（Desktop-First 寫法）*/
}

/* ── Mobile-First 斷點策略（推薦）── */

/* 基礎樣式：針對手機，沒有媒體查詢 */
.layout {
  display: flex;
  flex-direction: column;  /* 手機：垂直堆疊 */
  gap: 1rem;
}

/* 平板：640px 以上 */
@media (min-width: 640px) {
  .layout {
    flex-direction: row;     /* 平板：橫向排列 */
    flex-wrap: wrap;
  }
}

/* 桌面：1024px 以上 */
@media (min-width: 1024px) {
  .layout {
    gap: 2rem;
  }
  .sidebar {
    display: block;          /* 桌面才顯示側欄 */
  }
}

/* ── 常見 CSS 斷點參考 ── */
/* sm:  640px  — 大型手機、小型平板  */
/* md:  768px  — 平板               */
/* lg:  1024px — 小型桌面           */
/* xl:  1280px — 標準桌面           */
/* 2xl: 1536px — 大型桌面、超寬螢幕  */

/* ── 其他媒體條件 ── */
@media (prefers-color-scheme: dark) {
  :root { --bg: #0d1117; --text: #e6edf3; } /* 深色模式 */
}
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }  /* 停止動畫 */
}
05
CSS 自訂屬性（變數）
Design Tokens

CSS 自訂屬性（Custom Properties）讓你在 :root 定義全域變數，在整個樣式表中重複使用。當需要修改主色時，只需改一個地方，所有用到這個變數的地方都會自動更新。

這個機制也可以搭配媒體查詢實現深色模式：在 @media (prefers-color-scheme: dark) 中重新定義顏色變數，不需要重寫所有樣式規則，只需要覆寫變數值。

CSS — 自訂屬性（Design Tokens）
/* ── 定義設計 Token ── */
:root {
  /* 顏色 */
  --color-bg:       #ffffff;
  --color-text:     #1a1a2e;
  --color-primary:  #2563eb;
  --color-muted:    #6b7280;

  /* 間距 */
  --space-sm:  0.5rem;
  --space-md:  1rem;
  --space-lg:  2rem;
  --space-xl:  4rem;

  /* 字體 */
  --font-sans:  'DM Sans', sans-serif;
  --font-mono:  'JetBrains Mono', monospace;

  /* 圓角 */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 16px;
}

/* ── 深色模式自動切換 ── */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg:   #0d1117;
    --color-text: #e6edf3;
    /* 只需改變數值，其他樣式自動更新 */
  }
}

/* ── 使用變數 ── */
body {
  background: var(--color-bg);
  color: var(--color-text);
  font-family: var(--font-sans);
}

.card {
  padding: var(--space-lg);
  border-radius: var(--radius-md);
  background: var(--color-bg);
}

WEEK 02 PROJECT
履歷頁面
CSS 美化版

為第一週的 HTML 履歷加上樣式：設計配色系統、使用 Flexbox 排列導覽列、使用 Grid 排列技能卡片，並加入響應式斷點讓手機版正常顯示。

完成步驟

建立 style.css 並設定全域重設
新建 style.css，在 HTML <head> 用 <link rel="stylesheet" href="style.css"> 引入。先寫全域重設：*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }。設定 :root 顏色變數，至少包含背景色、文字色、主色調。
設定排版基礎：字體與間距
為 body 設定 font-family（從 Google Fonts 選一款）、line-height（建議 1.6–1.8）、background、color。建立 .container 類別：max-width: 900px; margin: 0 auto; padding: 0 1.5rem; 控制內容寬度。
用 Flexbox 排列 header 導覽列
讓 header 使用 Flexbox：左側放名字（flex: 1），右側放導覽連結（display: flex; gap: 1.5rem;）。設定 position: sticky; top: 0; 讓 header 固定在頁面頂部。加上 backdrop-filter: blur() 模糊效果（選修）。
用 Grid 排列技能卡片
把技能清單改成卡片樣式，每個 <li> 改為卡片外觀（padding、border、border-radius）。容器使用 display: grid; grid-template-columns: repeat(auto-fill, minmax(150px, 1fr)); gap: 1rem; 自動響應欄數。
設計工作經歷區塊
為每個 <article> 工作經歷加上樣式：左側加一條彩色邊框（border-left: 3px solid var(--color-primary)），搭配適當的 padding 和 margin。時間標籤使用 display: inline-block 設計成標籤樣式。
美化表單
為 input 和 textarea 設定統一樣式：width: 100%; padding: 0.6rem 1rem; border: 1px solid #ccc; border-radius: 4px;。加上 :focus 偽類樣式讓輸入框獲取焦點時有視覺回饋（border-color 改變）。為 button 設計配色和 hover 效果。
加入響應式媒體查詢
在 CSS 最底部加入媒體查詢。手機版（max-width: 768px）：讓 header 變成縱向堆疊、調小字體大小、增加各區塊的 padding。測試方式：Chrome DevTools 按 Ctrl+Shift+M 進入手機模擬模式。
加分項：Hover 動效與過渡
為導覽連結、技能卡片、按鈕加上 transition: all 0.2s ease; 和 :hover 樣式，讓互動更順滑。嘗試為卡片加上 :hover { transform: translateY(-4px); box-shadow: ... } 的浮起效果。
