核心概念 01

HTML 文件結構

每個 HTML 頁面都有固定的基本骨架。理解這個骨架，你就理解了瀏覽器如何解讀你的文件。

01
DOCTYPE 宣告
<!DOCTYPE html>

DOCTYPE 不是 HTML 標籤，它是一段給瀏覽器的指令，告訴瀏覽器這個文件是 HTML5 格式。它必須放在文件的第一行，在所有其他內容之前。

如果省略 DOCTYPE，瀏覽器會進入「怪異模式」（Quirks Mode），以舊版的不一致方式渲染頁面，導致很多難以預期的樣式問題。

NOTE 在 HTML4 時代，DOCTYPE 是一段很長的宣告字串。HTML5 大幅簡化為 <!DOCTYPE html>，這也是現代網頁開發的標準。
02
三大核心區塊
html / head / body

<html> 是整個文件的根元素，所有內容都在它裡面。lang 屬性告訴瀏覽器與螢幕閱讀器這個頁面使用的語言，對可及性非常重要，中文頁面應設為 zh-TW。

<head> 包含頁面的中繼資訊（metadata），這些內容不會顯示在頁面上，但對瀏覽器、搜尋引擎非常重要：文件編碼、頁面標題、CSS 引用等都放在這裡。

<body> 包含頁面上所有可見的內容：文字、圖片、連結、表單⋯⋯使用者看到的一切都在 body 裡。

HTML — 基本文件骨架
<!-- 每個 HTML 頁面的最小骨架 -->
<!DOCTYPE html>
<html lang="zh-TW">

<head>
  <!-- 告訴瀏覽器使用 UTF-8 編碼，支援中文 -->
  <meta charset="UTF-8">

  <!-- 讓手機版面正常縮放 -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- 顯示在瀏覽器分頁與搜尋結果的標題 -->
  <title>陳小明的個人履歷</title>
</head>

<body>
  <!-- 所有可見內容放這裡 -->
  <h1>Hello, World!</h1>
</body>

</html>
VS CODE 快速建立骨架 在 VS Code 中，新建一個 .html 檔案後，輸入 ! 再按 Tab 或 Enter，就會自動產生完整的 HTML5 骨架。這個功能叫做 Emmet，是前端開發者每天都在用的效率工具。
核心概念 02

語意化標籤

語意化標籤（Semantic HTML）讓標籤本身傳達內容的意義，而不僅僅是視覺呈現。這對 SEO、可及性和程式碼可讀性都有重大影響。

02
為什麼語意化很重要？
Semantic vs Non-semantic

想像你看到一份文件，裡面全部用相同字體、相同大小寫著所有內容——沒有標題、沒有段落，只是一大段文字。你很難快速理解哪裡是重點。HTML 語意化就是解決這個問題。

搜尋引擎用語意標籤理解你的內容結構，決定搜尋排名。螢幕閱讀器靠語意標籤幫助視障使用者導覽頁面。其他開發者閱讀你的程式碼時，語意標籤讓意圖一目瞭然。

常見錯誤 很多初學者習慣用 <div> 包裝所有東西，雖然不會出錯，但這樣的程式碼對搜尋引擎和輔助技術毫無意義。養成使用語意標籤的習慣，是成為專業前端的第一步。
標籤語意使用時機
<header>頁首 / 區塊標頭網站頂部導覽列，或一個 article 的標題區塊
<nav>導覽區網站主選單、分頁導覽、麵包屑導覽
<main>主要內容每個頁面只能有一個，包裝頁面的核心內容
<section>主題區塊頁面中有獨立主題的區塊，通常有標題
<article>獨立內容單元部落格文章、新聞、論壇貼文等可獨立存在的內容
<aside>附屬內容側邊欄、相關文章連結、補充說明
<footer>頁尾 / 區塊頁尾網站底部版權資訊，或 article 的作者資訊
<figure>獨立媒體內容包裝圖片、圖表，搭配 figcaption 使用
<time>時間日期或時間，搭配 datetime 屬性提供機器可讀格式
<address>聯絡資訊作者或組織的聯絡方式
HTML — 語意化頁面結構範例
<body>
  <header>
    <nav>
      <a href="/">首頁</a>
      <a href="/about">關於我</a>
    </nav>
  </header>

  <main>
    <section>
      <h1>關於我</h1>
      <p>前端開發者，熱愛創造使用者體驗。</p>
    </section>

    <section>
      <h2>工作經歷</h2>
      <article>
        <h3>前端工程師 @ 某公司</h3>
        <time datetime="2022-01">2022 年 1 月</time> — 至今
      </article>
    </section>
  </main>

  <footer>
    <p>© 2025 陳小明</p>
  </footer>
</body>
核心概念 03

文字內容標籤

03
標題層級
h1 — h6

HTML 提供六個層級的標題，從 <h1> 到 <h6>。這不只是文字大小的差異，更代表了內容的層次結構，就像文章的章節、小節、段落標題。

重要規則：每個頁面只應有一個 h1，作為整個頁面的主標題。不要為了讓文字變大就跳過層級（例如 h1 直接跳 h3），這會破壞文件結構並影響 SEO。

常見錯誤 不要用標題標籤來控制文字大小。標題層級應反映內容的語意層次，視覺大小應交給 CSS 處理。
03
行內強調標籤
strong / em / mark

<strong> 代表重要性強調，視覺上通常顯示為粗體。螢幕閱讀器會以不同語調朗讀。使用時機：警告文字、關鍵詞彙。

<em> 代表語氣強調，視覺上通常為斜體，類似於說話時特別加重的語氣。

注意：<b> 和 <i> 只是視覺上的粗體、斜體，沒有語意。在現代開發中，樣式控制應交給 CSS，標籤只負責語意。

HTML — 文字標籤實例
<!-- 段落與強調 -->
<p>
  這個職位<strong>需要 3 年以上經驗</strong>，
  並且<em>必須熟悉 React</em>。
</p>

<!-- 無序清單（技能清單）-->
<h3>技能清單</h3>
<ul>
  <li>HTML / CSS</li>
  <li>JavaScript</li>
</ul>

<!-- 有序清單（步驟流程）-->
<h3>開發步驟</h3>
<ol>
  <li>了解需求</li>
  <li>設計原型</li>
  <li>開發與測試</li>
</ol>

<!-- 引用區塊 -->
<blockquote>
  <p>先讓它能用，再讓它好用，最後讓它快。</p>
  <footer>— 程式開發諺語</footer>
</blockquote>
核心概念 04

連結與媒體

04
超連結 <a>
Anchor Element

href 屬性是連結的目標地址，可以是：外部網址（https://...）、相對路徑（../about.html）、頁面內錨點（#section-id）、email（mailto:）或電話（tel:）。

target="_blank" 讓連結在新分頁開啟。使用時請務必同時加上 rel="noopener noreferrer"——這是一個安全性最佳實踐，防止目標頁面通過 window.opener 存取你的頁面。

絕對路徑 vs 相對路徑 絕對路徑：完整 URL，如 https://example.com/page.html，適合連結到其他網站。相對路徑：相對於當前檔案的位置，如 ./about.html 或 ../images/photo.jpg，適合連結到同一網站的其他頁面。
04
圖片 <img>
Image Element

src 屬性指定圖片來源，alt 屬性提供替代文字。alt 屬性不是選填的——當圖片無法載入時顯示，更重要的是讓螢幕閱讀器能描述圖片給視障使用者。

如果圖片是純裝飾性的（對理解內容沒有幫助），使用 alt=""（空字串），螢幕閱讀器會直接略過它。如果圖片有資訊意義，請撰寫具體描述性的 alt 文字。

width 和 height 屬性可以告訴瀏覽器圖片的尺寸，讓瀏覽器在圖片載入前就保留空間，避免頁面跳動（Cumulative Layout Shift）。

HTML — 連結與圖片
<!-- 外部連結，在新分頁開啟 -->
<a href="https://github.com/username"
   target="_blank"
   rel="noopener noreferrer">
  我的 GitHub
</a>

<!-- 頁面內錨點跳轉 -->
<a href="#contact">聯絡我</a>

<!-- 有意義的圖片 -->
<img
  src="./images/profile.jpg"
  alt="陳小明的個人照片"
  width="200"
  height="200"
>

<!-- 圖片 + 說明文字 -->
<figure>
  <img src="chart.png" alt="2024 前端技術使用率統計圖">
  <figcaption>圖一：2024 年前端技術使用率</figcaption>
</figure>
核心概念 05

表單基礎

05
表單結構
form / input / label

<form> 是表單的容器，action 屬性指定提交後送往的 URL，method 決定使用 GET 或 POST 方式送出。

label 與 input 的關聯極其重要：使用 for 屬性與 input 的 id 對應，讓點擊 label 文字時，焦點自動跳到對應的 input。這對鍵盤使用者和螢幕閱讀器非常關鍵。

type 屬性決定 input 的類型，不同 type 在行動裝置上會觸發不同鍵盤，並有內建的基本驗證：email 會驗證 @ 符號，number 只接受數字，required 屬性讓欄位成為必填。

HTML — 聯絡表單
<form action="/contact" method="POST">

  <!-- for="id" 對應 input 的 id="id" -->
  <label for="name">姓名</label>
  <input type="text" id="name" name="name" required>

  <label for="email">電子郵件</label>
  <input type="email" id="email" name="email" required>

  <label for="message">訊息</label>
  <textarea id="message" name="message" rows="5"></textarea>

  <button type="submit">送出</button>

</form>
type 值用途行動裝置鍵盤
text一般文字輸入一般鍵盤
email電子郵件，自動驗證格式含 @ 符號的鍵盤
tel電話號碼數字鍵盤
number數字，可設定 min / max / step數字鍵盤
password密碼，輸入內容隱藏顯示一般鍵盤
checkbox多選核取方塊—
radio單選按鈕（同 name 群組互斥）—
date日期選擇器日期滾輪
file檔案上傳—

QUIZ 02 / 03
關於 <img> 的 alt 屬性，以下哪個說法是正確的？
Aalt 屬性是選填的，可以不寫
B裝飾性圖片應該使用 alt=""（空字串）
Calt 文字只有在圖片載入失敗時才有用
Dalt 文字越長越詳細越好
CORRECT裝飾性圖片設定 alt="" 告訴螢幕閱讀器「這個圖片沒有資訊意義，跳過它」。如果省略 alt 屬性，螢幕閱讀器可能會讀出檔案名稱，造成混亂的使用體驗。

QUIZ 03 / 03
以下哪種寫法最符合 label 與 input 的最佳實踐？
A<label for="email">信箱</label> <input id="email" type="email">
B<p>信箱</p> <input type="email">
C<label>信箱</label> <input type="email">
D<input type="email" placeholder="信箱">
CORRECT透過 for 與 id 的對應關聯 label 和 input，讓點擊 label 文字時焦點自動跳到 input。只用 placeholder 而不用 label 是常見的可及性錯誤——placeholder 消失後使用者可能忘記這個欄位的用途。
TRY AGAIN注意 label 的 for 屬性和 input 的 id 屬性之間的對應關係。

動手做：個人履歷頁面

將本週所學應用在一個真實場景中。這個專案會在第二週繼續加上 CSS 樣式，所以這週的重點是正確的語意結構，不需要在意視覺效果。

WEEK 01 PROJECT
個人履歷頁面
（純 HTML 版本）

建立一個只用 HTML 的個人履歷頁面，包含個人資訊、技能清單、工作/學習經歷與聯絡表單。不需要任何 CSS，重點在於正確的語意結構。

完成步驟

建立 HTML 文件骨架
新建 index.html，加入完整的 DOCTYPE、html（含 lang）、head（含 charset、viewport、title）、body 結構。用 VS Code 的 Emmet 快捷鍵 ! + Tab 快速完成。
規劃語意化外層結構
在 body 中加入：<header>（你的名字與職稱）、<main>（主要內容）、<footer>（版權資訊）。main 內用多個 <section> 區分不同區塊，每個 section 給予有意義的 id。
填入個人資訊區塊
在 header 或第一個 section 加入：<h1> 你的名字、<p> 簡短自介、一個 <img>（使用任意圖片 URL，記得設定有意義的 alt）、<address> 聯絡資訊。
加入技能與經歷
技能清單使用 <ul><li>。工作/學習經歷每筆使用一個 <article>，包含 <h3> 職稱/學校名稱、<time datetime="YYYY-MM"> 時間區間、<p> 描述文字。
加入導覽與社群連結
在 header 加入 <nav>，包含指向各 section id 的頁面內錨點連結。加入 GitHub、LinkedIn 等外部連結，記得使用 target="_blank" rel="noopener noreferrer"。
加入聯絡表單
在最後一個 section 加入聯絡表單，包含：姓名（type="text"）、信箱（type="email"）、訊息（textarea）。每個欄位都要有正確關聯的 label，並加上 required 屬性。
驗證與測試
使用 W3C HTML Validator 驗證是否有語法錯誤。打開瀏覽器開發者工具（F12）確認 Elements 面板的結構。嘗試只用 Tab 鍵導覽整個頁面，確認焦點順序正確。
