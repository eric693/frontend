02 — useEffect

什麼是副作用？

React 元件的核心工作是「根據 state/props 渲染畫面」。 但程式中有許多操作不屬於這個純粹的渲染過程——它們會影響元件外部的世界，這類操作統稱為副作用（Side Effects）。

資料請求
fetch / axios
向 API 拿資料、讀取資料庫，結果會改變 state 進而觸發重繪
訂閱 / 事件監聽
addEventListener
監聽 WebSocket、scroll、resize 事件，需要在元件消失時移除
計時器
setTimeout / setInterval
定期更新時間、輪詢資料；忘記清除會造成記憶體洩漏
直接操作 DOM
document.title / focus()
改變頁面標題、讓輸入框自動聚焦、整合第三方 JS 套件
Local Storage
localStorage.setItem()
持久化使用者設定（主題、語言偏好），同步至瀏覽器儲存
Log / 分析
console / 追蹤事件
頁面瀏覽記錄、GA 事件追蹤，在元件掛載或路由變更時觸發
UE
useEffect 基本語法
effect 函式 + 依賴陣列

useEffect(effect, deps) 接受兩個參數： effect 函式（要執行的副作用邏輯）與依賴陣列（決定何時重新執行）。 React 會在 DOM 更新完成、畫面繪製後才執行 effect，確保不阻塞渲染。

JSX — useEffect 基本結構 
import { useState, useEffect } from 'react'

function MovieDetail({ movieId }) {
  const [movie, setMovie] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    // ① effect 函式：這裡寫副作用邏輯
    setLoading(true)

    fetch(`https://api.example.com/movies/${movieId}`)
      .then(res => res.json())
      .then(data => {
        setMovie(data)
        setLoading(false)
      })

    // ② 依賴陣列：movieId 改變時重新執行 effect
  }, [movieId])

  if (loading) return <p>載入中...</p>
  return <h2>{movie?.title}</h2>
}
NOTE useEffect 是在畫面繪製後非同步執行的，不會阻塞瀏覽器渲染。 這和 class 元件的 componentDidMount 略有不同（後者在 commit 階段同步執行）。 若需要在 DOM 更新後立即同步讀取尺寸，應使用 useLayoutEffect。
03 — Lifecycle

生命週期與 useEffect 對照

React 16.8 前，副作用要分散在 class 元件的三個生命週期方法中。 useEffect 統一了這些邏輯，讓「相關的程式碼住在一起」。

Mount（掛載）
元件首次渲染到 DOM
class: componentDidMount
hook: useEffect(() => {}, [])
→
Update（更新）
props / state 發生變化
class: componentDidUpdate
hook: useEffect(() => {}, [dep])
→
Unmount（卸載）
元件從 DOM 移除
class: componentWillUnmount
hook: useEffect 回傳清除函式
useEffect deps 寫法 執行時機 對應 class 生命週期
無第二參數 每次重新渲染後都執行 componentDidUpdate（每次）
[]（空陣列） 只在掛載後執行一次 componentDidMount
[dep1, dep2] 掛載後 + dep 有變化時 componentDidMount + componentDidUpdate（帶條件）
回傳清除函式 下次 effect 執行前 + 卸載時 componentWillUnmount
04 — Dependency Array

依賴陣列深入解析

依賴陣列是 useEffect 最容易出錯的地方。 遺漏依賴會導致 effect 拿到過期的值（stale closure），過度依賴則造成無限迴圈。

無限迴圈陷阱
// ❌ 錯誤：在 effect 內更新 count
// 且把 count 列為依賴 → 無限迴圈
useEffect(() => {
  setCount(count + 1)
}, [count])
正確做法
// ✅ 使用 updater function
// 不需把 count 列為依賴
useEffect(() => {
  setCount(prev => prev + 1)
}, [])
JSX — 依賴陣列完整範例 
function SearchResults({ query, pageSize }) {
  const [results, setResults] = useState([])
  const [page, setPage] = useState(1)

  // ✅ query 或 pageSize 改變 → 重新搜尋並重設頁碼
  useEffect(() => {
    setPage(1)           // 重設到第 1 頁
    fetchResults(query, pageSize, 1).then(setResults)
  }, [query, pageSize]) // ← 列出所有用到的外部值

  // ✅ page 改變 → 只重新抓該頁資料
  useEffect(() => {
    if (page === 1) return  // 避免初始化時重複執行
    fetchResults(query, pageSize, page).then(setResults)
  }, [page])             // ← query/pageSize 已在上面的 effect 處理

  // ⚠️ ESLint 的 exhaustive-deps 規則會提示遺漏的依賴
  // 請安裝 eslint-plugin-react-hooks 來避免這類錯誤
}
COMMON MISTAKE 物件和函式每次渲染都會產生新的參考值，將它們直接放進依賴陣列會導致 effect 每次都重新執行。 解決方案：用 useMemo 穩定物件，用 useCallback 穩定函式，或重新設計資料流。
05 — Cleanup

清除副作用（Cleanup）

如果 effect 建立了訂閱、計時器或事件監聽，就必須在 effect 函式中 回傳一個清除函式。React 會在下次執行 effect 之前，以及元件卸載時，呼叫這個清除函式。

CL
使用 AbortController 取消 fetch
React 18 嚴格模式下 effect 執行兩次的標準解法

在 React 18 的 StrictMode 開發環境下，effect 會執行兩次（mount → unmount → mount）， 用來幫助你發現遺漏的 cleanup。正確的 fetch 寫法必須搭配 AbortController 來取消上一次未完成的請求。

JSX — AbortController cleanup 
useEffect(() => {
  const controller = new AbortController()

  async function loadMovie() {
    try {
      const res = await fetch(
        `/api/movies/${movieId}`,
        { signal: controller.signal }   // 傳入 signal
      )
      const data = await res.json()
      setMovie(data)
    } catch (err) {
      if (err.name !== 'AbortError') setError(err.message)
      // AbortError 是正常行為，不需要處理
    }
  }

  loadMovie()

  return () => controller.abort()  // cleanup：取消請求
}, [movieId])
JSX — 計時器與事件監聽的 cleanup 
// ① setInterval cleanup
useEffect(() => {
  const id = setInterval(() => {
    setTime(new Date())
  }, 1000)
  return () => clearInterval(id)   // 清除計時器
}, [])

// ② window 事件監聽 cleanup
useEffect(() => {
  const handleResize = () => setWidth(window.innerWidth)
  window.addEventListener('resize', handleResize)
  return () => window.removeEventListener('resize', handleResize)
}, [])

// ③ WebSocket cleanup
useEffect(() => {
  const ws = new WebSocket('wss://api.example.com/live')
  ws.onmessage = e => setMessages(prev => [...prev, e.data])
  return () => ws.close()             // 關閉 WebSocket 連線
}, [])
06 — Custom Hooks

自訂 Hook

當多個元件共用相同的 stateful 邏輯時，可以將它抽取成一個 自訂 Hook（名稱必須以 use 開頭）。 自訂 Hook 本質上就是一個呼叫其他 Hooks 的普通函式。

JSX — useFetch：通用資料請求 Hook 
// hooks/useFetch.js
import { useState, useEffect } from 'react'

export function useFetch(url) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    if (!url) return
    const controller = new AbortController()
    setLoading(true)
    setError(null)

    fetch(url, { signal: controller.signal })
      .then(res => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`)
        return res.json()
      })
      .then(json => { setData(json); setLoading(false) })
      .catch(err => {
        if (err.name !== 'AbortError') {
          setError(err.message)
          setLoading(false)
        }
      })

    return () => controller.abort()
  }, [url])

  return { data, loading, error }
}

// 在任意元件中使用：
function MovieCard({ id }) {
  const { data, loading, error } = useFetch(`/api/movies/${id}`)
  if (loading) return <Skeleton />
  if (error)   return <ErrorMsg msg={error} />
  return <h2>{data.title}</h2>
}
JSX — useDebounce：防抖搜尋 Hook 
// hooks/useDebounce.js
import { useState, useEffect } from 'react'

export function useDebounce(value, delay = 500) {
  const [debouncedValue, setDebouncedValue] = useState(value)

  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay)
    return () => clearTimeout(timer)  // 使用者繼續輸入時，清除計時器
  }, [value, delay])

  return debouncedValue
}

// 使用範例：只在使用者停止輸入 500ms 後才發出請求
function SearchBar() {
  const [query, setQuery] = useState('')
  const debouncedQuery = useDebounce(query, 500)

  // 只有 debouncedQuery 改變時才觸發 API 請求
  const { data } = useFetch(
    debouncedQuery ? `/api/search?q=${debouncedQuery}` : null
  )

  return (
    <input
      value={query}
      onChange={e => setQuery(e.target.value)}
      placeholder="搜尋電影..."
    />
  )
}
BEST PRACTICE 將自訂 Hook 放在 src/hooks/ 目錄下，每個 Hook 一個檔案（如 useFetch.js、useDebounce.js）。 這樣的組織方式讓 Hook 容易測試，也方便在不同專案間共用。
07 — useContext

useContext 與全域狀態

當資料需要跨越多個元件層級時，一層一層傳遞 props 會很麻煩，這稱為 Props Drilling。 Context 提供了一種讓元件直接「訂閱」資料來源的方式，不需要手動傳遞。

PROPS DRILLING vs CONTEXT
❌ PROPS DRILLING
App（持有 theme）
Layout（只傳遞 theme）
Sidebar（只傳遞 theme）
Button（真正用到 theme）
✅ CONTEXT
ThemeProvider（提供 theme）
Layout
Sidebar
Button（直接訂閱）
JSX — 建立 ThemeContext（完整流程） 
// contexts/ThemeContext.jsx
import { createContext, useContext, useState, useEffect } from 'react'

// ① 建立 Context（設定預設值，通常供測試使用）
const ThemeContext = createContext({
  theme: 'dark',
  toggleTheme: () => {}
})

// ② Provider 元件：持有 state，提供給子樹
export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState(() =>
    localStorage.getItem('theme') ?? 'dark'
  )

  // 同步至 localStorage，並套用 CSS class
  useEffect(() => {
    localStorage.setItem('theme', theme)
    document.documentElement.dataset.theme = theme
  }, [theme])

  const toggleTheme = () =>
    setTheme(t => t === 'dark' ? 'light' : 'dark')

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}

// ③ 自訂 Hook：讓使用方更簡潔，並加入防呆
export function useTheme() {
  const ctx = useContext(ThemeContext)
  if (!ctx) throw new Error('useTheme 必須在 ThemeProvider 內使用')
  return ctx
}

// ④ 在任意深度的元件中使用
function ThemeToggleButton() {
  const { theme, toggleTheme } = useTheme()
  return (
    <button onClick={toggleTheme}>
      {theme === 'dark' ? '🌙 深色' : '☀️ 淺色'}
    </button>
  )
}
⚡
Context 的效能考量
避免不必要的重新渲染

Context value 變化時，所有訂閱該 Context 的元件都會重新渲染。 若 Context 包含頻繁更新的資料（如滑鼠座標），效能會受到影響。

解決策略： ① 將頻繁更新與不常變動的資料拆成不同 Context（如 ThemeContext 與 UserContext）。 ② 用 useMemo 穩定 value 物件，避免每次渲染都建立新物件。 ③ 大型應用考慮使用 Zustand 或 Redux Toolkit 等狀態管理方案。

JSX — useMemo 穩定 Context value 
import { useMemo } from 'react'

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('dark')
  const toggleTheme = useCallback(() =>
    setTheme(t => t === 'dark' ? 'light' : 'dark')
  , [])

  // ✅ 只有 theme 改變時才建立新物件
  const value = useMemo(() => ({ theme, toggleTheme }), [theme, toggleTheme])

  return (
    <ThemeContext.Provider value={value}>
      {children}
    </ThemeContext.Provider>
  )
}

09 — Weekly Project

本週專案

PROJECT / WEEK 09
CineSearch — 電影搜尋頁
串接 TMDB（The Movie Database）公開 API，實作即時搜尋、分頁瀏覽、電影詳細資訊， 並加入深色 / 淺色主題切換，完整應用本週所有 Hook 技術。
申請 TMDB API Key
前往 themoviedb.org 免費申請帳號並取得 API Key。在專案根目錄建立 .env 檔，加入 REACT_APP_TMDB_KEY=你的key。記得將 .env 加入 .gitignore。
建立 ThemeContext
在 src/contexts/ThemeContext.jsx 建立 Provider，管理 theme 狀態並同步至 localStorage。在 App.jsx 用 <ThemeProvider> 包住整個應用，並輸出 useTheme 自訂 Hook。
實作 useFetch 與 useDebounce Hook
在 src/hooks/ 目錄建立兩個自訂 Hook：useFetch 封裝帶 AbortController 的 fetch 邏輯；useDebounce 實作 500ms 防抖，避免使用者每按一個鍵都發出請求。
SearchBar 元件與即時搜尋
建立 SearchBar 元件，使用 useDebounce 處理輸入值，再用 useFetch 向 /search/movie 端點發送請求。顯示骨架屏（Skeleton）作為載入中的佔位效果。
MovieGrid 與 MovieCard 元件
將搜尋結果以卡片網格呈現，每張卡片顯示電影海報（poster_path）、片名、評分與上映年份。加入圖片載入失敗的 fallback 圖片處理。
MovieDetail 彈出面板
點擊卡片後在右側滑入詳細資訊面板，使用獨立的 useEffect 根據所選電影 ID 請求詳細資料。同時更新 document.title 為當前電影名稱，離開時清除回預設值。
錯誤處理與空狀態設計
針對「網路錯誤」、「API 回傳 400/401/404」、「搜尋結果為空」三種情況分別顯示對應的提示訊息與圖示，讓使用者體驗完整。
主題切換整合
在 Header 加入切換按鈕，使用 useTheme 取得 toggleTheme。透過 CSS Variables 實作深色 / 淺色主題（在 :root[data-theme="light"] 覆寫變數），確保整個頁面平滑過渡。
