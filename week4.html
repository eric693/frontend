02 — Variables & Types

變數宣告與資料型別

JavaScript 是動態型別語言，變數本身沒有型別，但值有型別。ES6 之後，我們應優先使用 const， 需要重新賦值時才用 let，盡量避免 var。

V
var vs let vs const
作用域（Scope）與可變性的根本差異

var 是函式作用域（function-scoped），存在 提升（Hoisting） 行為， 且可以重複宣告，容易造成預期外的 bug。現代 JavaScript 幾乎不再使用它。

let 是區塊作用域（block-scoped），可以重新賦值但不能重複宣告， 也有提升但進入 暫時性死區（Temporal Dead Zone），存取前會拋出 ReferenceError。

const 同樣是區塊作用域，宣告後 不能重新賦值，但如果值是物件或陣列， 其內部屬性仍可修改（因為 const 鎖定的是「綁定」，不是「值」）。

✗ 避免使用 var
var count = 0
var count = 5   // 重複宣告，不報錯

for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i) // 全部印出 3！
  }, 100)
}
✓ 使用 let / const
const MAX = 100   // 不會改變的值
let count = 0     // 會變動的值
count++           // ✓ 可以

for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i) // 0, 1, 2 ✓
  }, 100)
}

03 — Functions

函式的三種形式

函式是 JavaScript 的一等公民（First-Class Citizen），可以被當作值傳遞、儲存在變數中、作為參數傳入或從函式回傳。 理解三種函式寫法的差異是掌握 JS 的關鍵。

F1
函式宣告（Function Declaration）
有 Hoisting，可在宣告前呼叫

用 function 關鍵字開頭的函式宣告會被提升到作用域頂部， 因此可以在宣告前就呼叫它。適合用在需要被大量呼叫的工具函式。

JavaScript — 函式宣告 
// 可以在宣告前呼叫（Hoisting）
console.log(greet("Eric")) // "Hello, Eric!"

function greet(name) {
  return `Hello, ${name}!`
}

// 帶預設參數（Default Parameters, ES6）
function createUser(name, role = "user", active = true) {
  return { name, role, active }
}
createUser("Eric")
// { name: "Eric", role: "user", active: true }

// 其餘參數（Rest Parameters）
function sum(...numbers) {
  return numbers.reduce((acc, n) => acc + n, 0)
}
sum(1, 2, 3, 4) // 10
F2
函式表達式（Function Expression）
沒有 Hoisting，只能在宣告後使用

將函式賦值給一個變數。沒有 Hoisting，適合需要根據條件決定要用哪個函式的場景， 也常用來建立 IIFE（立即呼叫函式表達式）。

JavaScript — 函式表達式 
// 函式表達式
const double = function(n) {
  return n * 2
}

// 根據條件選擇函式
const format = isMetric
  ? function(val) { return `${val} km` }
  : function(val) { return `${val} miles` }

// IIFE — 立即執行，建立私有作用域
const counter = (function() {
  let count = 0  // 外部無法存取
  return {
    increment: () => ++count,
    get: () => count
  }
})()
F3
箭頭函式（Arrow Function）
ES6 語法糖，不綁定自己的 this

箭頭函式語法更簡潔，並且不擁有自己的 this， 而是繼承外層的 this。這讓它非常適合用在回呼函式（callback）中， 例如 map、filter、事件監聽等場景。

單一參數可省略括號，單行 return 可省略大括號與 return 關鍵字。 但如果要直接回傳物件字面值，必須用小括號包住 () => ({...})。

JavaScript — 箭頭函式 
// 完整寫法
const add = (a, b) => {
  return a + b
}

// 省略 return（隱式回傳）
const add = (a, b) => a + b

// 單一參數，省略括號
const double = n => n * 2

// 回傳物件時需用小括號
const toObj = name => ({ name, active: true })

// 高階函式（Higher-Order Function）
const applyTwice = (fn, x) => fn(fn(x))
applyTwice(double, 3) // 12

// this 的差異
const obj = {
  name: "Eric",
  regularFn: function() {
    return this.name  // "Eric" ✓
  },
  arrowFn: () => {
    return this.name  // undefined（this 指向外層）
  }
}
💡 PURE FUNCTIONS 好的函式應盡量是純函式（Pure Function）：相同輸入永遠產生相同輸出， 且不修改外部狀態（無副作用）。這讓程式碼更可預測、更容易測試。
04 — Arrays

陣列與常用方法

陣列是 JavaScript 最常用的資料結構。ES5 之後引入的 map、filter、reduce 等高階方法讓我們能用函數式風格撰寫更簡潔、更安全的資料處理程式碼。

JavaScript — 陣列基礎 
// 建立陣列
const fruits = ["apple", "banana", "cherry"]
const nums = [1, 2, 3, 4, 5]
const mixed = [1, "hello", true, { id: 1 }]

// 基本操作
fruits[0]              // "apple"
fruits.length          // 3
fruits.push("date")    // 末端新增，回傳新長度
fruits.pop()           // 移除並回傳最後一個
fruits.unshift("fig") // 開頭新增
fruits.shift()         // 移除並回傳第一個

// splice — 刪除 / 插入（會修改原陣列）
fruits.splice(1, 1)           // 從索引 1 刪除 1 個
fruits.splice(1, 0, "mango") // 在索引 1 插入

// slice — 取出片段（不修改原陣列）
nums.slice(1, 3)   // [2, 3]
nums.slice(-2)    // [4, 5]

// Spread 展開運算子（ES6）
const newFruits = [...fruits, "kiwi"] // 複製並新增
const combined = [...nums, ...[6, 7]]  // 合併兩陣列

高階陣列方法（Higher-Order Methods）

.map()
arr.map(fn) → 新陣列
對每個元素執行 fn，回傳由結果組成的新陣列，長度與原陣列相同。
[1,2,3].map(n => n * 2) // [2,4,6]
.filter()
arr.filter(fn) → 新陣列
保留讓 fn 回傳 true 的元素，回傳過濾後的新陣列。
[1,2,3,4].filter(n => n > 2) // [3,4]
.reduce()
arr.reduce(fn, init) → 單一值
將陣列「摺疊」成單一值，acc 是累計器，init 是初始值。
[1,2,3].reduce((a,n) => a+n, 0) // 6
.forEach()
arr.forEach(fn) → undefined
對每個元素執行 fn，不回傳值，僅用於產生副作用。
arr.forEach(n => console.log(n))
.find()
arr.find(fn) → 元素或 undefined
回傳第一個讓 fn 為 true 的元素，找不到回傳 undefined。
[{id:1},{id:2}].find(o=>o.id===2)
.findIndex()
arr.findIndex(fn) → index 或 -1
回傳第一個符合條件的元素索引，找不到回傳 -1。
[10,20,30].findIndex(n=>n===20) // 1
.some() / .every()
arr.some(fn) → boolean
some：至少一個符合即 true；every：全部符合才 true。
[1,2,3].some(n=>n>2) // true
.includes()
arr.includes(val) → boolean
判斷陣列是否包含某值（使用 === 嚴格比較）。
["a","b","c"].includes("b") // true
JavaScript — 實際應用：Todo 資料處理 
const todos = [
  { id: 1, text: "學習 JavaScript", done: true  },
  { id: 2, text: "完成作業",         done: false },
  { id: 3, text: "運動 30 分鐘",    done: false },
]

// 只取未完成項目
const active = todos.filter(t => !t.done)

// 將 todo 轉換為 HTML 字串陣列
const html = todos
  .map(t => `<li class="${t.done ? 'done' : ''}">${t.text}</li>`)
  .join("")

// 計算已完成數量
const doneCount = todos.filter(t => t.done).length

// 切換完成狀態（不修改原陣列）
const toggleTodo = (todos, id) =>
  todos.map(t => t.id === id ? { ...t, done: !t.done } : t)

// 刪除指定 id（不修改原陣列）
const deleteTodo = (todos, id) =>
  todos.filter(t => t.id !== id)
📌 不可變性（IMMUTABILITY） 優先使用回傳新陣列的方法（map、filter、slice、spread） 而非直接修改原陣列（push、splice），這是函數式程式設計的核心原則， 也是 React 要求的寫法。

05 — Objects

物件與解構賦值

物件（Object）是 JavaScript 的基礎資料結構，幾乎所有的複雜資料都用物件來表示。 ES6 引入的解構賦值、簡寫屬性、計算屬性名稱等語法，讓操作物件更加優雅。

JavaScript — 物件基礎 
// 物件字面值
const user = {
  id: 1,
  name: "Eric",
  email: "eric@example.com",
  address: {           // 巢狀物件
    city: "Taipei",
    country: "Taiwan"
  },
  greet() {           // 方法簡寫（ES6）
    return `Hi, I'm ${this.name}`
  }
}

// 兩種存取方式
user.name           // "Eric"（點記法）
user["email"]       // "eric@example.com"（括號記法）
user.address.city  // "Taipei"（鏈式存取）

// 新增 / 修改 / 刪除屬性
user.age = 25            // 新增
user.name = "Eric Chen"  // 修改
delete user.email        // 刪除

// ES6 簡寫屬性（變數名 = 屬性名）
const name = "Eric", age = 25
const u = { name, age }  // { name: "Eric", age: 25 }

// 計算屬性名稱（Computed Property Names）
const key = "status"
const obj = { [key]: "active" }  // { status: "active" }
D
解構賦值（Destructuring Assignment）
從物件或陣列中快速提取值

解構賦值是 ES6 最實用的語法之一，讓我們能從物件或陣列中直接提取需要的值到獨立變數， 避免重複寫 obj.prop。

物件解構可以設定預設值、使用別名，也可以進行巢狀解構。 搭配函式參數使用時，能讓函式介面更清晰。

JavaScript — 解構賦值 
const user = {
  id: 1, name: "Eric", age: 25,
  address: { city: "Taipei", zip: "100" }
}

// 基本物件解構
const { name, age } = user
console.log(name) // "Eric"

// 別名（Alias）
const { name: userName } = user
console.log(userName) // "Eric"

// 預設值
const { role = "user", name: uName } = user
console.log(role)  // "user"（user 物件沒有 role）

// 巢狀解構
const { address: { city } } = user
console.log(city) // "Taipei"

// 其餘屬性（Rest in Object）
const { id, ...rest } = user
console.log(rest) // { name, age, address }

// 陣列解構
const [first, second, ...others] = [1, 2, 3, 4]
// first=1, second=2, others=[3,4]

// 函式參數解構（最常見用法）
function renderUser({ name, age, role = "user" }) {
  return `${name} (${age}) - ${role}`
}
JavaScript — 物件實用方法 
const person = { name: "Eric", age: 25, city: "Taipei" }

// Object.keys / values / entries
Object.keys(person)    // ["name", "age", "city"]
Object.values(person)  // ["Eric", 25, "Taipei"]
Object.entries(person) // [["name","Eric"],["age",25],["city","Taipei"]]

// 用 entries 遍歷物件
Object.entries(person).forEach(([key, val]) => {
  console.log(`${key}: ${val}`)
})

// Object.assign — 合併物件（淺複製）
const defaults = { theme: "dark", lang: "zh-TW" }
const config = Object.assign({}, defaults, { theme: "light" })
// { theme: "light", lang: "zh-TW" }

// Spread 更簡潔（推薦）
const config2 = { ...defaults, theme: "light" }

// 選擇性鏈結（Optional Chaining, ES2020）
const city = user?.address?.city  // 不會因 undefined 報錯

// Nullish Coalescing（??, ES2020）
const name = user.name ?? "Guest" // 只有 null/undefined 才用預設
📌 淺複製 vs 深複製 Spread 運算子 { ...obj } 只能做淺複製（Shallow Copy）， 巢狀物件仍是同一個參考。若需要深複製，可用 JSON.parse(JSON.stringify(obj))（有限制） 或 structuredClone(obj)（現代瀏覽器支援）。
06 — Quiz

隨堂測驗

測試你對本週核心概念的理解。點擊選項後查看解釋。

QUESTION 01 / 04
以下程式碼的輸出結果是什麼？
const arr = [1, 2, 3]; arr.push(4); console.log(arr);
A 報錯，因為 const 宣告的變數不能被修改
B [1, 2, 3, 4]
C [1, 2, 3]，因為 const 不允許改變
D undefined
正確！ const 鎖定的是「綁定（binding）」， 不是值本身。陣列是物件，arr 存的是記憶體位址，這個位址沒有改變， 所以 push 完全合法。只有 arr = [5, 6] 這種重新賦值才會報錯。
QUESTION 02 / 04
下列哪個陣列方法不會修改原陣列？
A push()
B splice()
C map()
D sort()
正確！ map() 回傳一個全新的陣列，不修改原陣列。 push() 在末端新增元素、splice() 可插入/刪除、sort() 就地排序， 三者都會直接改變原陣列。這個區別在 React 狀態管理中極為重要。
QUESTION 03 / 04
以下程式碼的輸出是什麼？
console.log(typeof null)
A "null"
B "undefined"
C "object"
D "boolean"
正確！ typeof null === "object" 是 JavaScript 的歷史遺留 bug， 從 1995 年第一版就存在，為了不破壞現有程式碼而保留至今。 要正確判斷一個值是否為 null，應使用嚴格相等：value === null。
QUESTION 04 / 04
以下哪個寫法可以將陣列 [1,2,3,4,5] 轉換成所有偶數的平方和？
A [1,2,3,4,5].map(n => n*n).filter(n => n%2===0).reduce((a,n)=>a+n)
B [1,2,3,4,5].filter(n => n%2===0).map(n => n*n).reduce((a,n)=>a+n, 0)
C [1,2,3,4,5].reduce((a,n) => a+n*n, 0)
D [1,2,3,4,5].forEach(n => n%2===0 ? n*n : 0)
正確！ 正確順序：先 filter 篩出偶數 → 再 map 取平方 → 最後 reduce 加總。 選項 A 的錯誤在於先取平方再篩偶數，會把奇數的平方（如 9、25）誤篩出來。 reduce 第二參數 0 確保空陣列時不報錯。
