02 — Design Philosophy

Utility-First：思維的轉換

傳統 CSS 的做法是先為元素命名，再撰寫樣式。 Tailwind 反過來：直接在 HTML 上組合低階的工具類，不需要發明 class 名稱。

傳統 CSS 做法
<!-- HTML -->
<div class="card">
  <h2 class="card-title">片名</h2>
  <p class="card-desc">簡介</p>
</div>

/* CSS */
.card {
  background: #1a1a1a;
  border-radius: 12px;
  padding: 1.5rem;
  box-shadow: 0 4px 20px rgba(0,0,0,.3);
}
.card-title {
  font-size: 1.25rem;
  font-weight: 700;
  color: #fff;
  margin-bottom: .5rem;
}
.card-desc { color: #aaa; }
Tailwind 做法
<!-- HTML — 不需要額外的 CSS 檔 -->
<div class="bg-zinc-900 rounded-xl
       p-6 shadow-xl shadow-black/30">
  <h2 class="text-xl font-bold
         text-white mb-2">片名</h2>
  <p class="text-zinc-400">簡介</p>
</div>

// 零 CSS 切換！只需改 class 名稱
// 所有樣式直接可見，無需來回跳檔
// 建置後只輸出實際使用到的 CSS
JIT
JIT（Just-In-Time）編譯引擎
Tailwind v3 預設啟用，打包體積接近零

Tailwind v3 的 JIT 引擎會掃描你的程式碼，找出所有用到的 class， 只把這些 class 對應的 CSS 規則輸出到最終檔案中。 開發中即時生成、不需要 PurgeCSS，生產環境的 CSS 通常只有 5–20 KB（gzip 後）。

JIT 還解鎖了任意值語法：w-[342px]、text-[#ff6b35]、grid-cols-[1fr_2fr_1fr]—— 只要在方括號內填入任意 CSS 值，JIT 就會即時生成對應的工具類。

MINDSET SHIFT 剛接觸 Tailwind 時，看到 HTML 上一串 class 會感到不適應——這很正常。 關鍵在於：你不再需要為樣式命名，也不再需要在 HTML 和 CSS 之間來回切換。 設計決策直接體現在標記語言上，可讀性反而更高。
03 — Installation

安裝與設定

以 Vite + React 專案為例，完整走一遍 Tailwind CSS v3 的安裝流程。

Terminal — 安裝步驟 
# 1. 建立 Vite + React 專案（若尚未建立）
npm create vite@latest my-portfolio -- --template react
cd my-portfolio

# 2. 安裝 Tailwind 及其 peer dependencies
npm install -D tailwindcss postcss autoprefixer

# 3. 產生設定檔
npx tailwindcss init -p
# 生成 tailwind.config.js 與 postcss.config.js
tailwind.config.js — 設定掃描範圍 
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,jsx,ts,tsx}",  // ← JIT 掃描這些檔案
  ],
  darkMode: 'class',              // 使用 class 控制深色模式
  theme: {
    extend: {
      // 自訂設計 token 放這裡（後面章節說明）
    },
  },
  plugins: [],
}
src/index.css — 引入 Tailwind 指令 
/* 三個指令順序不能錯 */
@tailwind base;        /* 重置瀏覽器預設樣式（Preflight）*/
@tailwind components;  /* 元件層（供 @apply 抽取）*/
@tailwind utilities;   /* 所有工具類 */
VS CODE 推薦 安裝官方擴充套件 Tailwind CSS IntelliSense，能在 class 屬性中自動補全、預覽顏色、 顯示對應的 CSS 值。這是提升開發效率最重要的工具之一。
常見錯誤 content 路徑寫錯是最常見的問題。如果樣式沒有生效，先確認： ① 路徑是否涵蓋所有使用 Tailwind class 的檔案、 ② class 名稱是否完整（不能動態拼接如 "text-" + color，JIT 無法掃描到動態字串）。

07 — Component Patterns

元件設計與類名管理

當同樣的 Tailwind class 組合在多處重複出現時，有三種主流做法來管理它。 選擇哪種取決於重複的程度與動態性的需求。

01
策略一：React 元件抽象（推薦）
最符合 React 思維，同時保有 props 動態化的能力

在 React 中，重複的 UI 應該抽成元件，而不是抽成 CSS class。 這是 Tailwind 在元件化框架中最正確的使用姿勢。 元件封裝了 class 的組合，同時可以透過 props 控制動態變化。

JSX — Button 元件：用 props 控制變體 
// components/Button.jsx
const variants = {
  primary:   'bg-amber-400 text-zinc-900 hover:bg-amber-300',
  secondary: 'bg-zinc-800 text-white hover:bg-zinc-700',
  outline:   'border border-amber-400 text-amber-400 hover:bg-amber-400/10',
  ghost:     'text-zinc-400 hover:text-white hover:bg-zinc-800',
}

const sizes = {
  sm: 'px-3 py-1.5 text-sm',
  md: 'px-5 py-2.5 text-base',
  lg: 'px-7 py-3.5 text-lg',
}

export function Button({
  children,
  variant = 'primary',
  size = 'md',
  disabled,
  className = '',
  ...props
}) {
  return (
    <button
      className={`
        ${variants[variant]}
        ${sizes[size]}
        font-semibold rounded-lg
        transition-all duration-200
        active:scale-95
        disabled:opacity-40 disabled:cursor-not-allowed
        ${className}
      `.trim()}
      disabled={disabled}
      {...props}
    >
      {children}
    </button>
  )
}

// 使用方式
<Button variant="primary" size="lg">查看作品集</Button>
<Button variant="outline">聯絡我</Button>
<Button variant="ghost" disabled>Coming Soon</Button>
02
策略二：@apply（純 CSS 複用）
適合需要在 CSS 中復用樣式，或整合第三方 HTML 的情況

@apply 讓你在 CSS 中提取工具類成為語意化的 class。 適用場景：① Markdown 渲染的內容（無法加 class）、② 第三方 UI 覆蓋、 ③ 需要在多個框架中共用的樣式。在 React 專案中，通常優先考慮元件抽象。

CSS — @apply 提取元件樣式 
/* src/index.css */
@tailwind base;
@tailwind components;

@layer components {
  .btn-primary {
    @apply bg-amber-400 text-zinc-900 px-5 py-2.5
           font-semibold rounded-lg
           hover:bg-amber-300 active:scale-95
           transition-all duration-200;
  }

  /* Markdown prose 樣式（需安裝 @tailwindcss/typography） */
  .prose-custom {
    @apply prose prose-invert
           prose-headings:text-amber-400
           prose-a:text-amber-300;
  }
}

@tailwind utilities;
03
策略三：clsx / cn 工具函式
管理條件式 class 合併，避免字串拼接的混亂

當元件有多個 props 會影響 class 時，字串模板會變得雜亂。 clsx（或 shadcn/ui 常用的 cn 函式）提供更清晰的條件式 class 合併方式。

JSX — clsx 條件式 class 合併 
import { clsx } from 'clsx'              // npm install clsx
import { twMerge } from 'tailwind-merge'   // npm install tailwind-merge

// cn：先合併條件，再去除 Tailwind class 衝突
function cn(...inputs) {
  return twMerge(clsx(inputs))
}

// 使用範例
function Badge({ label, active, size }) {
  return (
    <span className={cn(
      'inline-flex items-center rounded-full font-medium',
      active
        ? 'bg-amber-400/20 text-amber-300 ring-1 ring-amber-400/50'
        : 'bg-zinc-800 text-zinc-400',
      size === 'sm' && 'px-2 py-0.5 text-xs',
      size === 'md' && 'px-3 py-1 text-sm',
    )}>
      {label}
    </span>
  )
}
08 — Dark Mode

深色模式實作

Tailwind 支援兩種深色模式策略：media（跟隨系統設定）與 class（手動切換）。 作品集網站通常選擇 class 模式，讓使用者自行控制。

策略 說明 適用場景
darkMode: 'media' 自動跟隨 prefers-color-scheme 系統設定 不需要手動切換按鈕的簡單網站
darkMode: 'class' 在 <html> 上加 dark class 來啟用 需要切換按鈕、並記憶使用者偏好的應用
darkMode: ['class', '[data-theme="dark"]'] v3.4+：用自訂屬性選擇器取代 class 整合現有主題系統（如本課程前幾週用的方法）
JSX — 深色模式切換完整實作 
// hooks/useTheme.js
import { useState, useEffect } from 'react'

export function useTheme() {
  const [isDark, setIsDark] = useState(() =>
    localStorage.getItem('theme') !== 'light'  // 預設深色
  )

  useEffect(() => {
    document.documentElement.classList.toggle('dark', isDark)
    localStorage.setItem('theme', isDark ? 'dark' : 'light')
  }, [isDark])

  return { isDark, toggle: () => setIsDark(d => !d) }
}

// components/ThemeToggle.jsx
import { useTheme } from '../hooks/useTheme'

export function ThemeToggle() {
  const { isDark, toggle } = useTheme()
  return (
    <button
      onClick={toggle}
      className="p-2 rounded-lg
                 bg-zinc-800 hover:bg-zinc-700
                 dark:bg-zinc-700 dark:hover:bg-zinc-600
                 transition-colors duration-200"
      aria-label="切換主題"
    >
      {isDark ? '☀️' : '🌙'}
    </button>
  )
}

{/* 在元件中使用 dark: 修飾符 */}
<div className="
  bg-white dark:bg-zinc-900
  text-zinc-900 dark:text-zinc-100
  border border-zinc-200 dark:border-zinc-700
">
09 — Custom Theme

自訂主題（tailwind.config.js）

tailwind.config.js 是設計系統的核心。 在 theme.extend 中新增設計 Token，讓整個專案共用統一的顏色、字型、間距規範。

tailwind.config.js — 完整自訂主題範例 
export default {
  content: ["./index.html", "./src/**/*.{js,jsx}"],
  darkMode: 'class',
  theme: {
    extend: {
      // 自訂顏色 palette（可搭配 CSS Variables）
      colors: {
        brand: {
          50:  '#fffbeb',
          400: '#fbbf24',
          500: '#f59e0b',
          900: '#78350f',
        },
        surface: 'rgb(var(--color-surface) / <alpha-value>)',
      },

      // 自訂字型
      fontFamily: {
        sans:    ['Outfit', 'system-ui', 'sans-serif'],
        display: ['Playfair Display', 'serif'],
        mono:    ['JetBrains Mono', 'monospace'],
      },

      // 自訂動畫
      keyframes: {
        fadeUp: {
          '0%':   { opacity: '0', transform: 'translateY(20px)' },
          '100%': { opacity: '1', transform: 'translateY(0)' },
        },
        shimmer: {
          '0%':   { backgroundPosition: '-200% 0' },
          '100%': { backgroundPosition: '200% 0' },
        }
      },
      animation: {
        fadeUp:  'fadeUp .5s ease forwards',
        shimmer: 'shimmer 2s infinite linear',
      },

      // 延伸斷點
      screens: {
        '2xl': '1536px',
        '3xl': '1920px',
      },
    },
  },
  plugins: [
    require('@tailwindcss/typography'),   // Markdown 排版
    require('@tailwindcss/forms'),         // 表單重置樣式
    require('@tailwindcss/aspect-ratio'), // 比例容器
  ],
}
DESIGN TOKEN TIP 在 theme.extend 中新增的項目會合併到預設主題中， 直接使用 theme（不用 extend）則會完全覆蓋預設值，通常應使用 extend。 建議將品牌顏色命名為 brand 或專案名稱，而非直接覆蓋 blue、gray 等預設色系。

將第 10 週用 React Router 建立的作品集網站，全面移植至 Tailwind CSS。 重新設計視覺系統、加入完整的 RWD 響應式支援、深色模式切換， 並完善各頁面的微互動細節，達到可對外展示的完成度。
安裝 Tailwind 並遷移設定
在現有的作品集專案中安裝 Tailwind CSS v3、clsx、tailwind-merge。設定 tailwind.config.js 的 content 路徑與 darkMode: 'class'，並在 index.css 引入三行 Tailwind 指令，同時移除原有的自訂 CSS 檔案。
建立設計系統（tailwind.config.js）
在 theme.extend 中定義品牌色（如 brand-400、brand-500）、自訂字型（display、mono）與動畫（fadeUp、shimmer）。建立 src/lib/utils.js，輸出 cn 工具函式供全專案使用。
重構基礎 UI 元件
建立 components/ui/ 目錄，製作可重用的 Button、Badge、Card、Tag 元件，每個元件都支援 variant 和 size props。確保所有元件支援深色模式（dark: 修飾符）。
首頁 Hero Section RWD 重設計
重新設計 Hero：手機版垂直排列，桌機版水平兩欄（照片 + 文字）。使用 animate-fadeUp 加入進場動畫，headings 使用 Playfair Display 搭配自訂色，加入 CTA 按鈕組（主按鈕 + 次要按鈕）。
作品集頁面：卡片網格 + Hover 效果
實作響應式卡片網格（手機1欄 → 平板2欄 → 桌機3欄）。每張卡片使用 group + group-hover 實作「滑入顯示 overlay + 縮放圖片」的懸停效果，加入技術標籤（Badge 元件）與連結按鈕。
技能頁面：進度條與分類展示
設計技能分類卡片（前端、工具、設計），使用 Tailwind 的 w-0 → w-[80%] 搭配 transition-all 製作動畫進度條（Intersection Observer 觸發）。技術 icon 使用 react-icons。
導覽列 + 主題切換整合
重做 Navbar：使用 backdrop-blur-md 和 bg-white/80 dark:bg-zinc-900/80 製作毛玻璃效果，加入 useTheme Hook 與 ThemeToggle 按鈕。手機版使用 Hamburger Menu，點擊後滑入全寬 Drawer（Tailwind transition）。
聯絡表單 + Skeleton Loading
安裝 @tailwindcss/forms，重新美化聯絡表單，加入 focus 光環、錯誤狀態（ring-red-500）與成功狀態。為作品集卡片製作 Skeleton 載入效果（使用 animate-shimmer）。
