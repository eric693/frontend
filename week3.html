核心概念 01

CSS 自訂屬性（變數）

CSS 自訂屬性不只是「把顏色存起來」，它是建立整套設計系統的基礎，也是實現動態主題切換（深色模式）最乾淨的方式。

01
宣告與使用
--variable-name / var()

CSS 自訂屬性以 兩個連字號（--）開頭宣告，用 var() 函式取用。慣例上，全域變數宣告在 :root 選擇器中，讓整份樣式表都能存取。

var() 接受第二個參數作為備用值（fallback）：var(--color-primary, #2563eb)——當變數未定義時使用 #2563eb。這讓程式碼更健壯，尤其在元件化開發中很有用。

CSS 變數遵循作用域與繼承規則：在某個元素上宣告的變數，只有該元素及其後代可以使用。你可以在子元素中覆寫父元素定義的變數，實現區域主題效果。

CSS — 自訂屬性完整語法
/* ── 全域設計 Token ── */
:root {
  /* 顏色系統 */
  --color-bg:        #ffffff;
  --color-bg-alt:    #f8f9fa;
  --color-text:      #1a1a2e;
  --color-muted:     #6b7280;
  --color-primary:   #e8547a;
  --color-secondary: #a78bfa;
  --color-border:    #e5e7eb;

  /* 間距比例尺（4px base）*/
  --space-1: .25rem;   /* 4px  */
  --space-2: .5rem;    /* 8px  */
  --space-4: 1rem;     /* 16px */
  --space-8: 2rem;     /* 32px */
  --space-16: 4rem;    /* 64px */

  /* 圓角 */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 16px;
  --radius-full: 9999px; /* 膠囊形狀 */

  /* 陰影 */
  --shadow-sm:  0 1px 3px rgba(0,0,0,.1);
  --shadow-md:  0 4px 12px rgba(0,0,0,.12);
  --shadow-lg:  0 12px 32px rgba(0,0,0,.15);

  /* 過渡速度 */
  --transition-fast:   .15s ease;
  --transition-normal: .25s ease;
  --transition-slow:   .4s ease;
}

/* ── 在元素中使用變數 ── */
.card {
  background:    var(--color-bg);
  border:        1px solid var(--color-border);
  border-radius: var(--radius-md);
  padding:       var(--space-8);
  box-shadow:    var(--shadow-sm);
  transition:    box-shadow var(--transition-normal);
}
.card:hover {
  box-shadow: var(--shadow-lg);
}

/* ── 區域變數覆寫（主題島）── */
.card-featured {
  --color-bg:     #fff0f4; /* 只在這張卡片內有效 */
  --color-border: #e8547a;
  background: var(--color-bg);
}

/* ── 備用值（fallback）── */
.text {
  color: var(--color-brand, #e8547a); /* 未定義時用 #e8547a */
}

命名慣例 推薦使用 語意命名而非描述外觀：--color-primary 比 --color-pink 好，因為當你想把主色從粉紅改成藍色時，後者就失去意義了。設計系統通常分為三層：Primitive Token（原始值：--pink-500: #e8547a）、Semantic Token（語意值：--color-primary: var(--pink-500)）、Component Token（元件值：--btn-bg: var(--color-primary)）。
核心概念 02

Transition 過渡效果

Transition 是 CSS 動畫的入門磚——它讓屬性值的改變不是瞬間跳躍，而是平滑過渡。掌握它，你的 UI 就有了基本的生命感。

02
transition 語法拆解
property / duration / timing / delay

transition 由四個部分組成：要過渡的屬性、持續時間、緩動函數（timing function）、延遲時間。可以一次設定多個屬性，用逗號分隔。

使用 transition: all 雖然方便，但會監聽所有屬性的變化，效能較差且可能產生意外效果（例如 display 無法過渡）。推薦明確指定要過渡的屬性，如 transition: background-color .2s, transform .3s。

哪些屬性可以 transition？ 只有「數值型」屬性才能平滑過渡：顏色、尺寸、位置、透明度、transform 等。display: none → block、height: 0 → auto 這類無法數值插值的改變不能直接 transition（後者需要技巧處理）。

CSS — Transition 完整語法
/* ── 基本語法 ── */
.element {
  /* property | duration | timing-function | delay */
  transition: background-color .3s ease 0s;

  /* 多屬性（逗號分隔）*/
  transition:
    background-color .3s ease,
    transform .25s cubic-bezier(.34,1.56,.64,1),
    box-shadow .3s ease;
}

/* ── Timing Functions ── */
.btn { transition: transform .2s ease; }         /* 預設：先快後慢 */
.btn { transition: transform .2s linear; }       /* 等速 */
.btn { transition: transform .2s ease-in; }      /* 先慢後快（進場感）*/
.btn { transition: transform .2s ease-out; }     /* 先快後慢（自然感）*/
.btn { transition: transform .2s ease-in-out; }  /* 兩端慢中間快 */

/* cubic-bezier 自訂緩動 */
.btn { transition: transform .3s cubic-bezier(.34,1.56,.64,1); } /* 彈跳 */
.btn { transition: transform .4s cubic-bezier(.68,-.55,.27,1.55); } /* 彈性 */

/* ── 常見 hover 模式 ── */
.card {
  transform: translateY(0);
  box-shadow: var(--shadow-sm);
  transition: transform .25s ease, box-shadow .25s ease;
}
.card:hover {
  transform: translateY(-6px); /* 上浮 */
  box-shadow: var(--shadow-lg);
}

.link {
  text-decoration: none;
  background-image: linear-gradient(var(--color-primary),var(--color-primary));
  background-size: 0% 2px;         /* 初始：無底線 */
  background-position: bottom left;
  background-repeat: no-repeat;
  transition: background-size .3s ease;
}
.link:hover {
  background-size: 100% 2px;       /* hover：底線從左展開 */
}
Timing Function特性適合場景
ease預設，先快後慢，自然感大多數通用場景
linear等速，機械感進度條、旋轉 loading
ease-in先慢後快，加速感元素離開畫面
ease-out先快後慢，減速感元素進入畫面（最自然）
ease-in-out兩端慢，中間快需要對稱的動作
cubic-bezier()完全自訂四個控制點彈跳、彈性、個性化效果
核心概念 03

@keyframes 動畫

Transition 只能在兩個狀態之間過渡，@keyframes 讓你定義任意多個動畫階段，打造複雜的連續動作——旋轉、跳動、波浪、閃爍，全都在掌控之中。

03
@keyframes 定義動畫幀
from/to / 百分比

@keyframes 定義一個有名字的動畫序列，你可以在動畫的任意時間點（用百分比 0%–100% 或 from/to）設定 CSS 屬性值，瀏覽器自動在各幀之間插值。

定義好 @keyframes 後，用 animation 屬性將其套用到元素上。最重要的兩個參數是動畫名稱和持續時間。

transform 是動畫的最佳朋友：translateX/Y（位移）、scale（縮放）、rotate（旋轉）都用 GPU 加速，比直接改 top/left/width 效能好得多。opacity 也是 GPU 加速屬性。這兩類屬性是製作動畫的首選。

CSS — @keyframes 完整語法與常見動畫
/* ── 基本語法 ── */
@keyframes slideFadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* ── 多個關鍵幀 ── */
@keyframes bounce {
  0%, 100%   { transform: translateY(0); }
  40%        { transform: translateY(-20px); }
  60%        { transform: translateY(-10px); }
}

/* ── animation 屬性：6 個參數 ── */
.element {
  /* name | duration | timing | delay | iteration | direction */
  animation: slideFadeIn .6s ease-out 0s 1 normal;

  /* 簡寫（常用形式）*/
  animation: slideFadeIn .6s ease-out forwards;
}

/* ── 分開寫更清晰 ── */
.card {
  animation-name:            slideFadeIn;
  animation-duration:        .6s;
  animation-timing-function: ease-out;
  animation-delay:           0s;
  animation-iteration-count: 1;         /* infinite = 無限循環 */
  animation-direction:       normal;    /* alternate = 來回 */
  animation-fill-mode:       forwards; /* 停在最後一幀 */
  animation-play-state:      running;  /* paused = 暫停 */
}

/* ── 交錯動畫（stagger）── */
.item                { animation: slideFadeIn .5s ease-out both; }
.item:nth-child(1)   { animation-delay: 0s; }
.item:nth-child(2)   { animation-delay: .1s; }
.item:nth-child(3)   { animation-delay: .2s; }
.item:nth-child(4)   { animation-delay: .3s; }
/* 每個元素依序延遲，形成瀑布進場效果 */

/* ── 實用動畫庫 ── */
@keyframes spin      { to { transform: rotate(360deg); } }
@keyframes ping      { 75%,100% { transform: scale(2); opacity: 0; } }
@keyframes pulse     { 0%,100% { opacity: 1; } 50% { opacity: .5; } }
@keyframes shimmer   {
  0%   { background-position: -200% center; }
  100% { background-position: 200% center; }
}
核心概念 04

動畫控制與 fill-mode

animation-fill-mode 是動畫結束後「停在哪裡」的關鍵設定，理解它能解決「動畫播完後元素跳回原狀」的常見困惑。

04
animation-fill-mode
none / forwards / backwards / both

none（預設）：動畫播放前後，元素都維持原本的樣式。動畫結束後「跳回」原位。

forwards：動畫結束後，元素停留在最後一幀（to / 100%）的樣式。製作「滑入顯示」動畫時必用。

backwards：有 animation-delay 時，在延遲期間就套用第一幀（from / 0%）的樣式，避免延遲期間元素突然出現。

both：同時套用 forwards 和 backwards 的效果。交錯動畫（stagger）幾乎都需要用 both。

CSS — fill-mode 與 JavaScript 控制動畫
/* ── fill-mode 四個值 ── */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.enter-none      { animation: fadeIn .5s ease none; }
/* 播完後跳回 opacity:0，元素消失 */

.enter-forwards  { animation: fadeIn .5s ease forwards; }
/* 播完後停在 opacity:1，元素留在畫面上 ✓ */

.enter-both      { animation: fadeIn .5s ease .3s both; }
/* 延遲 .3s 前就套用 from 的樣式（opacity:0）*/
/* 結束後停在 to 的樣式（opacity:1）✓ 最常用 */

/* ── play-state：用 JS 暫停/繼續 ── */
.paused { animation-play-state: paused; }

/* JavaScript */
document.querySelector('.loader').style.animationPlayState = 'paused';

/* ── Scroll 觸發動畫（Intersection Observer）── */
.reveal {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity .6s ease, transform .6s ease;
}
.reveal.visible {  /* JS 加上 .visible 類別 */
  opacity: 1;
  transform: translateY(0);
}

/* JavaScript Intersection Observer */
const observer = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) e.target.classList.add('visible');
  });
}, { threshold: 0.1 });

document.querySelectorAll('.reveal').forEach(el => observer.observe(el));

/* ── 無障礙：尊重使用者的動畫偏好 ── */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration:   .01ms !important;
    animation-iteration-count: 1  !important;
    transition-duration:  .01ms !important;
  }
}
fill-mode效果使用場景
none播放前後都是元素本身的樣式循環動畫、不需要停留效果
forwards播放結束後停留在最後一幀一次性出場/入場動畫
backwards延遲期間套用第一幀樣式有 delay 且需要避免初始閃現
both同時套用 forwards + backwards有 delay 的交錯動畫，幾乎最通用
核心概念 05

深色模式

深色模式不是「把背景換成黑色」那麼簡單。這週你將學會用 CSS 變數搭配媒體查詢，優雅地實作自動跟隨系統設定的深色模式。

05
prefers-color-scheme
系統深色模式偵測

@media (prefers-color-scheme: dark) 會在使用者的作業系統設定為深色模式時生效。搭配 CSS 變數，你只需在這個媒體查詢中覆寫 :root 的顏色變數，不需要重寫任何元件的樣式——所有用了顏色變數的元素都會自動切換。

深色模式的設計原則：不要只是把顏色反轉。深色背景不應該是純黑（#000000），通常是帶一點色調的深灰（如 GitHub 的 #0d1117）。陰影在深色模式下效果不明顯，應改用描邊（border）來區分層次。

手動切換深色模式 除了跟隨系統，你也可以讓使用者手動切換。常見做法是在 <html> 或 <body> 標籤加上 data-theme="dark" 屬性，然後在 CSS 中用 [data-theme="dark"] 選擇器覆寫變數。用 JavaScript 切換這個屬性，就能實現手動切換按鈕。

CSS — 深色模式完整實作
/* ── 淺色模式（預設）── */
:root {
  --color-bg:      #ffffff;
  --color-bg-alt:  #f8f9fa;
  --color-text:    #1a1612;
  --color-muted:   #8a826f;
  --color-border:  #d8d0c4;
  --color-primary: #c4581a;
  --shadow:        0 4px 12px rgba(0,0,0,.1);
}

/* ── 自動跟隨系統設定 ── */
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg:      #0e0e0f;  /* 帶色調的深灰，非純黑 */
    --color-bg-alt:  #161618;
    --color-text:    #f0ede8;
    --color-muted:   #888888;
    --color-border:  #2a2a2e;
    --color-primary: #c8f06e;  /* 深色模式用更亮的主色 */
    --shadow:        0 4px 12px rgba(0,0,0,.4);
    /* 不需要改任何其他規則，所有元件自動更新 */
  }
}

/* ── 手動切換（JS 按鈕）── */
[data-theme="dark"] {
  --color-bg:   #0e0e0f;
  --color-text: #f0ede8;
  /* 其他變數... */
}

/* JavaScript 切換主題 */
const toggle = document.getElementById('themeToggle');
toggle.addEventListener('click', () => {
  const current = document.documentElement.dataset.theme;
  document.documentElement.dataset.theme =
    current === 'dark' ? '' : 'dark';
});

/* ── 加入過渡讓切換更平滑 ── */
body {
  transition: background-color .3s ease, color .3s ease;
  /* 注意：其他元件也需要加 transition 才會平滑過渡 */
}
深色模式設計原則 深色模式不是簡單反轉色調。幾個重要原則：（1）避免純黑背景（#000），使用 #0e0e0f 或 #0d1117 這類帶微弱色調的深灰；（2）降低飽和度，深色背景上鮮豔的顏色會過於刺眼；（3）陰影變透明度更高（rgba(0,0,0,.4) 而非 .1）；（4）邊框代替陰影做層次區分。
核心概念 06

完整 RWD 實戰策略

這週把響應式設計從概念推進到實戰——三個斷點、完整的縮放策略、以及最容易被忽略的排版細節。

06
常見 RWD 問題與解法
圖片溢出 / 文字過大 / 觸控目標過小

圖片溢出容器：永遠加上 img { max-width: 100%; height: auto; } 作為全域重設。這讓所有圖片最多和容器一樣寬，但不強制拉伸。

文字大小不縮放：使用 clamp() 函式讓文字在不同螢幕尺寸間流暢縮放：font-size: clamp(1.5rem, 3vw, 3rem) 意思是「最小 1.5rem，最大 3rem，中間按視窗寬度的 3% 計算」。

觸控目標過小：手機上按鈕和連結的最小可點擊面積應為 44px × 44px（Apple 人機介面指南建議）。按鈕 padding 要夠大，不要只讓文字本身可點擊。

測試方法 Chrome DevTools 按 Ctrl+Shift+M（Mac: Cmd+Shift+M）進入裝置模擬模式。選擇不同裝置預設值，或手動調整視窗寬度。同時注意「Responsive」模式下拖動邊框，觀察版面在各寬度下的變化。
✕ 常見錯誤
/* 固定像素寬度 */
.container { width: 1200px; }

/* 字體固定大小 */
h1 { font-size: 48px; }

/* 忘記圖片限制 */
/* (沒有 max-width: 100%) */

/* 按鈕太小 */
.btn { padding: 2px 8px; }
✓ 正確做法
/* 彈性最大寬度 */
.container { max-width: 1200px; width: 100%; }

/* 流體字體 */
h1 { font-size: clamp(2rem,5vw,3.5rem); }

/* 圖片永遠不溢出 */
img { max-width: 100%; height: auto; }

/* 觸控友善按鈕 */
.btn { padding: .75rem 1.5rem; }

隨堂練習

測試你的理解

三道題目，涵蓋本週的動畫與深色模式核心概念。

QUIZ 01 / 03
你製作了一個元素從 opacity: 0 淡入到 opacity: 1 的進場動畫，但播放結束後元素又消失了（跳回 opacity: 0）。應該加上哪個設定來讓它停留在畫面上？
Aanimation-direction: alternate
Banimation-iteration-count: infinite
Canimation-fill-mode: forwards
Danimation-play-state: paused
CORRECTanimation-fill-mode: forwards 讓動畫結束後停留在最後一幀的樣式（opacity: 1）。infinite 會讓動畫無限重播（每次都從 0 開始），不是你要的效果；alternate 讓動畫來回播放；paused 是暫停播放。
TRY AGAIN想想「動畫結束後的狀態」——哪個屬性控制動畫播完後元素停留的位置？
QUIZ 02 / 03
以下哪兩個 CSS 屬性最適合用來製作動畫？（因為它們使用 GPU 加速，效能最好）
Atransform 和 opacity
Bwidth 和 height
Ctop 和 left
Dmargin 和 padding
CORRECTtransform（位移、縮放、旋轉）和 opacity 是「合成屬性」（composited properties），瀏覽器可以直接交給 GPU 處理，不需要觸發 Layout（重排）或 Paint（重繪）。改變 width/height、top/left、margin 都會觸發 Layout，效能成本高出許多。
TRY AGAIN提示：哪些屬性的改變不需要重新計算頁面佈局（Layout）？
QUIZ 03 / 03
你用 CSS 變數實作了深色模式，淺色模式的主色是 --color-primary: #c4581a。在深色模式下，你應該如何覆寫它讓對比度更好？
A在每個元素的選擇器中重新設定 color: #f0a070
B在 @media (prefers-color-scheme: dark) { :root { --color-primary: #f0a070; } } 中覆寫變數
C使用 !important 強制覆寫
D為深色模式另外建立一份完整的 CSS 檔案
CORRECT在 @media (prefers-color-scheme: dark) 內的 :root 中覆寫 CSS 變數，是最乾淨的做法。所有使用了 var(--color-primary) 的元素都會自動更新，不需要動任何其他樣式。這正是 CSS 變數配合深色模式的威力所在。
TRY AGAINCSS 變數的核心優勢是什麼？想想只改一個地方就能讓所有元素同步更新的方法。

動手做：動效與深色模式

這週的任務是在前兩週的履歷頁面上，加入動畫效果和完整的深色模式支援，讓它從「靜態頁面」晉升為「有互動感的現代網頁」。

WEEK 03 PROJECT
動態履歷
深色模式版

為第二週的履歷加入：進場動畫（stagger effect）、hover 互動效果、自動深色模式，以及完整的三斷點響應式設計。

完成步驟

重整 CSS 變數系統
在 :root 中建立完整的設計 Token：顏色（至少 6 個）、間距比例尺（--space-1 到 --space-16）、圓角（--radius-sm/md/lg）、過渡速度（--transition-fast/normal）。確保所有樣式都改用變數，不再有硬編碼的顏色值。
加入深色模式
加入 @media (prefers-color-scheme: dark)，在其中的 :root 覆寫所有顏色變數。深色背景不要用 #000，改用 #0e0e0f 或 #111113。同時為 body 加上 transition: background-color .3s, color .3s 讓切換平滑。
頁面進場動畫
定義 @keyframes fadeIn（opacity 0→1，translateY 20→0）。為 header、hero 標題、hero 描述依序設定 animation: fadeIn .6s ease both，並用遞增的 animation-delay（0s / .15s / .3s / .45s）製造交錯進場效果。
Scroll Reveal 效果
為 section 元素加上 .reveal class，初始 opacity: 0; transform: translateY(30px)，有 .visible 時過渡到 opacity: 1; transform: translateY(0)。用 IntersectionObserver 監聽元素進入視窗時加上 .visible，threshold 設 0.1。
技能卡片 Hover 效果
為每個技能卡片加上 hover 效果：transition: transform .25s, box-shadow .25s，hover 時 transform: translateY(-6px); box-shadow: 0 12px 24px rgba(0,0,0,.12)。同時改變 border-color 為主色調，讓卡片「亮起來」。
導覽連結底線動效
用 background-image + background-size 技巧製作底線展開動效（程式碼參考概念 2 的範例）。從寬度 0% 2px hover 到 100% 2px，加上 transition: background-size .3s ease。
完整三斷點 RWD
加入手機（預設）、平板（640px+）、桌面（1024px+）三個斷點。確認：全局加了 img { max-width: 100%; }、標題使用 clamp()、按鈕 padding 足夠大（手機觸控友善）、沒有任何元素在手機版橫向溢出。
無障礙動畫設定
在 CSS 最底部加入 @media (prefers-reduced-motion: reduce)，在其中停用所有動畫和過渡（設為 .01ms）。在 macOS 的「系統設定 → 輔助使用 → 減少動態效果」中測試是否正常停用。

完成項目自評

所有硬編碼顏色都已改為 CSS 變數，:root 有完整的設計 Token
深色模式在系統設定切換時自動生效，顏色對比度良好
頁面載入時 header 和 hero 有交錯進場動畫（至少 3 個元素有不同 delay）
使用 IntersectionObserver 實作 Scroll Reveal，section 滾入時才顯示
技能卡片 hover 時有 transform + box-shadow 浮起效果，過渡平滑
導覽連結有底線展開動效（不使用預設底線，改用 background-size 技巧）
三個斷點都有效，手機版沒有橫向溢出，按鈕大小適合觸控
