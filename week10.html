核心概念 01

SPA 路由是什麼？

傳統網站每換一頁就向伺服器要一份新 HTML。SPA 只載入一次 HTML，之後的「換頁」完全在瀏覽器裡發生——這就是前端路由。

01
傳統 MPA vs 現代 SPA
Multi-Page App vs Single-Page App

傳統的多頁應用（MPA）：點擊連結 → 瀏覽器向伺服器發送請求 → 伺服器回傳完整 HTML → 頁面重新載入。每次換頁都有白屏閃爍，速度取決於網路。

單頁應用（SPA）：只有第一次載入時請求 HTML，之後的「換頁」完全在瀏覽器端發生——JavaScript 更新 URL（用 History API）、根據 URL 決定渲染哪個元件、切換速度接近即時。

React Router 的工作就是：監聽 URL 變化，根據 URL 決定渲染哪個 React 元件。它是 URL 和元件之間的對應表。

History API 瀏覽器的 History API 讓 JavaScript 可以改變 URL（history.pushState()）而不觸發頁面重整。React Router 在底層使用這個 API——這也是為什麼你需要在伺服器設定「所有路由都回傳 index.html」，否則直接訪問 /about 會得到 404。
01
BrowserRouter vs HashRouter
兩種路由模式的選擇

BrowserRouter（推薦）：使用真實的 URL 路徑，如 example.com/about。外觀乾淨，SEO 友好。需要伺服器配合設定（Vite dev server 已自動處理，部署到 Vercel/Netlify 也自動支援）。

HashRouter：URL 中有 #，如 example.com/#/about。# 後的部分不會被送到伺服器，因此不需要伺服器特別設定，適合靜態檔案伺服器或某些特殊環境。但 URL 不好看，也不利 SEO。

結論：在 Vite 開發環境和主流部署平台（Vercel、Netlify）使用 BrowserRouter。只有在無法設定伺服器時才考慮 HashRouter。

核心概念 02

安裝與基本設定

React Router v6 是目前的主流版本，API 設計比 v5 更直覺。學習 v6 的寫法，避免在網路上看到 v5 範例時混用。

02
核心元件：BrowserRouter、Routes、Route
v6 路由架構的三個支柱

BrowserRouter（簡稱 Router）是路由的提供者，需要包裹整個應用（通常放在 main.jsx 或 App.jsx 最外層）。它監聽瀏覽器 URL 的變化並提供路由上下文給所有子元件。

Routes 是路由規則的容器，會在所有子 Route 中找到最匹配當前 URL 的那一個並渲染它。v6 中的 Routes 自動只渲染最佳匹配，不需要像 v5 那樣加 exact 屬性。

Route 定義一條路由規則：path 是 URL 模式，element 是匹配時要渲染的 JSX 元素。

Link 取代 <a href>——它渲染成 <a> 標籤，但點擊時不會讓頁面重整，而是更新 URL 並觸發 React Router 重新渲染對應元件。NavLink 是 Link 的增強版，當路由匹配時自動加上 active 類別，適合導覽列高亮效果。

v5 寫法（已過時）
// react-router-dom v5
import { Switch, Route } from 'react-router-dom';

function App() {
  return (
    <Switch>
      {/* exact 避免 / 匹配所有 */}
      <Route exact path="/"
        component={HomePage} />
      <Route path="/about"
        component={AboutPage} />
    </Switch>
  );
}
v6 寫法（現代，推薦）
// react-router-dom v6
import { Routes, Route } from 'react-router-dom';

function App() {
  return (
    <Routes>
      {/* 不需要 exact，自動最佳匹配 */}
      <Route path="/"
        element={<HomePage />} />
      <Route path="/about"
        element={<AboutPage />} />
    </Routes>
  );
}

Link vs <a> 的本質差異 <a href="/about"> 讓瀏覽器向伺服器請求 /about 的 HTML，造成整頁重載。<Link to="/about"> 用 History API 更新 URL，觸發 React Router 更換元件，不向伺服器發請求、不重載頁面。在 React 應用中，站內連結一律用 Link；站外連結（指向其他網站）才用 <a>。
核心概念 03

動態路由參數

一個電商網站有上萬個商品頁面，不可能為每個商品定義一條路由。動態路由讓你用一條規則匹配所有這類 URL。

03
URL 解剖：param、query、hash
三種資訊傳遞方式

URL 可以攜帶三種不同形式的資訊，各有適用場景：

Route Param（路徑參數）：URL 路徑的一部分，如 /posts/42 中的 42。用 :id 定義，用 useParams() 取得。適合必要的、識別資源的資訊。

Query String（查詢字串）：URL 中 ? 後面的鍵值對，如 /search?q=react&page=2。用 useSearchParams() 取得。適合可選的、過濾/分頁資訊。

Hash（錨點）：URL 中 # 後的部分，如 /about#contact。用於頁面內錨點跳轉，不會送到伺服器。

import { useParams } from 'react-router-dom';

function ProjectDetail() {
  const { id, category } = useParams();
  // 存取 /projects/web/42 時：id="42", category="web"
  // useParams 回傳的值都是字串！數字要自行轉換
  const projectId = Number(id);

  // 常見用法：用 param 取得對應資料
  const [project, setProject] = useState(null);
  useEffect(() => {
    fetchProject(id).then(setProject);
  }, [id]); // id 是依賴項，路由改變時重新取資料

  if (!project) return <div>載入中...</div>;
  return <h1>{project.title}</h1>;
}

// ── 取得與設定 Query String ──
import { useSearchParams } from 'react-router-dom';

function SearchPage() {
  const [searchParams, setSearchParams] = useSearchParams();

  const query = searchParams.get('q');        // 取值
  const page  = searchParams.get('page') ?? '1'; // 預設值

  const handleSearch = (newQuery) => {
    setSearchParams({ q: newQuery, page: '1' }); // 更新 URL
    // URL 變為 /search?q=react&page=1
  };

  return (
    <div>
      <p>搜尋：{query}，第 {page} 頁</p>
      <button onClick={() => setSearchParams({ q: query, page: String(Number(page)+1) })}>
        下一頁
      </button>
    </div>
  );
}
核心概念 04

巢狀路由與 Outlet

大多數應用都有共用的 Layout（Header + Footer 不變，中間內容切換）。巢狀路由讓你優雅地實現這個模式，而不是在每個頁面元件中重複加 Header 和 Footer。

04
Outlet：子路由的插入點
共用 Layout 的核心機制

Outlet 是一個特殊元件，放在 Layout 元件中，作為「子路由內容的插入點」。當 URL 匹配到某個子路由時，React Router 把對應的子元件渲染到 Outlet 的位置。

這讓你可以定義一個 RootLayout 元件包含 Header 和 Footer，其中的 <Outlet /> 會根據 URL 自動替換成 HomePage、AboutPage 等子頁面內容。Header 和 Footer 永遠存在，不會因為切換頁面而重新掛載。

巢狀路由也可以多層——例如 Dashboard 有自己的子路由（/dashboard/analytics、/dashboard/users），每一層都可以有自己的 Layout 和 Outlet。

JSX — 巢狀路由與 Outlet
// ── Layout 元件 ──
import { Outlet, Link } from 'react-router-dom';

function RootLayout() {
  return (
    <div>
      <header>
        <nav>
          <Link to="/">首頁</Link>
          <Link to="/about">關於</Link>
        </nav>
      </header>

      <main>
        <Outlet />  {/* 子路由內容插入這裡 */}
      </main>

      <footer>© 2025</footer>
    </div>
  );
}

// ── App.jsx：定義巢狀路由 ──
function App() {
  return (
    <Routes>
      {/* 父路由：渲染 Layout */}
      <Route path="/" element={<RootLayout />}>
        {/* 子路由：渲染進 Outlet */}
        <Route index           element={<HomePage />} />  {/* index = 預設子路由 */}
        <Route path="about"   element={<AboutPage />} />
        <Route path="projects" element={<ProjectsPage />} />

        {/* 巢狀路由的巢狀路由 */}
        <Route path="dashboard" element={<DashboardLayout />}>
          <Route index                 element={<DashboardHome />} />
          <Route path="analytics"      element={<Analytics />} />
          <Route path="users"          element={<Users />} />
        </Route>

        <Route path="*" element={<NotFoundPage />} />
      </Route>
    </Routes>
  );
}
核心概念 05

程式化導航

有時候換頁不是使用者點擊 Link，而是由程式碼觸發——表單提交成功後跳轉、登出後回到首頁、條件判斷後自動跳轉。這需要 useNavigate()。

05
useNavigate：push vs replace
歷史記錄的管理策略

useNavigate() 回傳一個 navigate 函式，呼叫 navigate('/path') 就像使用者點了 Link。預設是 push 模式——把新路由推入瀏覽器歷史記錄，使用者可以按返回鍵回到之前的頁面。

replace 模式（navigate('/path', { replace: true })）：取代當前歷史記錄，而非新增一條。適合「不應該能返回的」跳轉——例如登入成功後跳轉到主頁，不希望使用者按返回鍵又回到登入頁。

navigate(-1) 等同瀏覽器的返回鍵；navigate(1) 等同前進鍵；navigate(-2) 後退兩步。這讓你可以用程式控制瀏覽器歷史的前進後退。

傳遞 state 給下一頁 navigate('/success', { state: { orderId: 42 } }) 可以把資料傳給下一頁，在目標頁面用 useLocation().state 取得。這適合傳遞「一次性的訊息」（如成功通知），不需要儲存在 URL 或 localStorage 中。

JSX — useNavigate 完整用法
import { useNavigate, useLocation } from 'react-router-dom';

// ── 基本程式化跳頁 ──
function LoginForm() {
  const navigate = useNavigate();

  const handleSubmit = async (e) => {
    e.preventDefault();
    await login(credentials);

    // 登入成功：用 replace，不讓使用者返回登入頁
    navigate('/dashboard', { replace: true });
  };
  return <form onSubmit={handleSubmit}>...</form>;
}

// ── 儲存「登入後要去哪」再跳轉 ──
function ProtectedPage() {
  const navigate = useNavigate();
  const location = useLocation();  // 取得當前位置資訊

  if (!isLoggedIn) {
    // 記住想去的頁面，登入後再跳轉回去
    navigate('/login', {
      replace: true,
      state: { from: location.pathname } // 傳遞當前路徑
    });
    return null;
  }
  return <div>受保護內容</div>;
}

// ── 在登入頁接收並使用 state ──
function LoginPage() {
  const navigate = useNavigate();
  const location = useLocation();
  const from = location.state?.from ?? '/'; // 登入前想去的頁面

  const handleLogin = async () => {
    await login();
    navigate(from, { replace: true }); // 回到原本要去的頁面
  };
}

navigate(-1);                         // 返回上一頁

核心概念 06

保護路由

有些頁面需要登入才能存取（儀表板、個人設定、管理後台）。保護路由是一個包裝元件，在渲染子路由之前先檢查使用者是否有權限。

06
PrivateRoute 設計模式
在渲染前進行權限檢查

PrivateRoute（或稱 RequireAuth）是一個包裝元件：如果使用者已登入，渲染 <Outlet />（或 children）讓路由正常進入；如果未登入，用 <Navigate /> 元件自動重新導向到登入頁。

<Navigate /> 是宣告式的版本——它是一個元件，渲染時自動觸發跳轉，效果等同呼叫 navigate()。在需要在 JSX return 中觸發跳轉時（而非事件處理器中）使用它。

最佳實踐：記住使用者原本要去的頁面（用 state: { from: location }），讓使用者登入後可以直接回到目標頁面，而非每次都回到首頁。這個細節大幅提升使用者體驗。

JSX — 完整保護路由實作
import { Navigate, Outlet, useLocation } from 'react-router-dom';

// ── 方法一：PrivateRoute 元件（推薦）──
function PrivateRoute() {
  const { user } = useAuth();       // 從 AuthContext 取得使用者狀態
  const location = useLocation();

  if (!user) {
    // 未登入：跳轉到登入頁，記住原本的路徑
    return <Navigate to="/login"
             state={{from: location}}
             replace />;
  }
  // 已登入：渲染子路由
  return <Outlet />;
}

// ── 在路由設定中使用 ──
function App() {
  return (
    <Routes>
      <Route path="/"      element={<RootLayout />}>
        <Route index        element={<HomePage />} />
        <Route path="login" element={<LoginPage />} />

        {/* 受保護的路由群組 */}
        <Route element={<PrivateRoute />}>
          <Route path="dashboard" element={<Dashboard />} />
          <Route path="profile"   element={<Profile />} />
          <Route path="settings"  element={<Settings />} />
        </Route>
      </Route>
    </Routes>
  );
}

// ── 登入頁：登入後跳回原本的頁面 ──
function LoginPage() {
  const navigate = useNavigate();
  const location = useLocation();
  const { login } = useAuth();
  const from = location.state?.from?.pathname ?? '/';

  const handleLogin = async (credentials) => {
    try {
      await login(credentials);
      navigate(from, { replace: true }); // 跳回原本要去的頁面
    } catch {
      setError('帳號或密碼錯誤');
    }
  };
  return <form>...</form>;
}

// ── 方法二：角色權限路由 ──
function RequireRole({ allowedRoles }) {
  const { user } = useAuth();
  if (!user) return <Navigate to="/login" replace />;
  if (!allowedRoles.includes(user.role))
    return <Navigate to="/403" replace />;
  return <Outlet />;
}

// 使用：只有 admin 可以存取
<Route element={<RequireRole allowedRoles={['admin']} />}>
  <Route path="admin" element={<AdminPanel />} />
</Route>

在 React 應用中，站內頁面之間的跳轉應該用哪個元件？
A<a href="/about"> ── 標準 HTML 連結
B<button onClick={() => window.location.href = '/about'}>
C<Link to="/about"> ── React Router 提供的連結元件
D<a href="/about" target="_blank">
CORRECT<Link> 使用 History API 更新 URL，不觸發頁面重整，React Router 監聽到 URL 變化後渲染對應元件——這就是 SPA 的精髓。選項 A 和 B 都會讓整個頁面重新載入，應用的 state 全部丟失，效果和傳統網站無異。
TRY AGAIN思考「SPA 換頁的核心原則」——換頁時頁面不能重整，所有的 React 狀態要保留。哪種方式能做到這點？
QUIZ 02 / 04
你定義了路由 /posts/:id，使用者訪問 /posts/hello-world。在元件中 useParams().id 的值是什麼？
A數字 0（找不到對應的數字 ID）
B字串 "hello-world"
Cundefined（只接受數字）
D路由不會匹配，渲染 404 頁面
CORRECTuseParams() 永遠回傳字串，不管 URL 中的值看起來是數字還是文字。:id 是通配符，匹配任何非 / 的字元。所以 /posts/hello-world、/posts/42、/posts/abc-123 都能匹配，id 值分別是對應的字串。需要數字時，自行用 Number(id) 或 parseInt(id) 轉換。
TRY AGAIN記住：useParams() 的回傳值型別是什麼？URL 是純文字，解析出來的值自然也是⋯⋯
QUIZ 03 / 04
Outlet 元件的作用是什麼？
A它是 React Router 內建的 404 頁面元件
B它用來在元件之間傳遞資料，類似 Context
C它是路由的出口，所有路由都必須放在 Outlet 裡面
D放在 Layout 元件中，標記「子路由元件應該渲染在這個位置」
CORRECT<Outlet /> 是巢狀路由的核心機制。你在 Layout 元件（如 RootLayout）中放一個 <Outlet />，React Router 會把當前匹配的子路由元件渲染到這個位置。Header 和 Footer 永遠存在（因為它們在 Layout 裡），只有 Outlet 的位置會根據 URL 改變。
TRY AGAINOutlet 和巢狀路由有關。想想「共用 Layout + 切換頁面內容」這個需求——Outlet 解決了什麼問題？
QUIZ 04 / 04
實作保護路由時，未登入的使用者訪問 /dashboard，你希望他們跳轉到 /login 後，登入成功再回到 /dashboard。最佳做法是什麼？
A用 localStorage 儲存目標路徑，登入後讀取並跳轉
B登入後永遠跳回首頁（/），讓使用者自己再導航
C用 navigate('/login', { state: { from: location } }) 傳遞來源頁面，登入後用 location.state?.from 取得並跳轉
D在 URL 加上 ?redirect=/dashboard，登入後解析 query string
CORRECT用 React Router 的 state 傳遞來源位置是最優雅的做法：navigate('/login', {'{'} state: {'{'} from: location {'}'} {'}'})，在登入頁用 useLocation().state?.from 取得來源路徑，登入成功後 navigate(from, {'{'} replace: true {'}'})。這不需要 localStorage，state 只在一次導航中存在，不會殘留。
TRY AGAINReact Router 的 navigate 函式可以傳遞額外的 state 給目標頁面，這個 state 在目標頁面用 useLocation() 取得。這比操作 localStorage 更乾淨。
本週實作

動手做：個人作品集網站

這週把前幾週的 React 技能整合起來，建立一個有多個頁面、巢狀路由和完整導覽的個人作品集網站。

WEEK 10 PROJECT
個人作品集
多頁面 React 應用

使用 React Router v6 建立真實的多頁面作品集網站，涵蓋共用 Layout、動態作品詳情頁、404 頁面、以及可選的身份驗證流程。

完成步驟

建立頁面結構與安裝 React Router
在 Week 08 的 Vite 專案基礎上，執行 npm install react-router-dom。建立 src/pages/ 目錄，新增：HomePage.jsx、AboutPage.jsx、ProjectsPage.jsx、ProjectDetailPage.jsx、ContactPage.jsx、NotFoundPage.jsx。每個頁面先放最小化的 JSX。
建立 RootLayout 與共用導覽列
新建 src/layouts/RootLayout.jsx，包含 <header>（用 NavLink 建立導覽，active 時高亮）、<main><Outlet /></main>、<footer>。在 main.jsx 或 App.jsx 設定 BrowserRouter，在 App.jsx 建立巢狀路由結構，RootLayout 作為父路由。
定義完整路由規則
在 App.jsx 中設定：/（index，HomePage）、/about（AboutPage）、/projects（ProjectsPage）、/projects/:id（ProjectDetailPage）、/contact（ContactPage）、*（NotFoundPage，自訂 404 頁面加返回首頁按鈕）。確認每條路由都能正常訪問。
建立作品集資料與 ProjectsPage
新建 src/data/projects.js，匯出一個作品陣列（至少 4 個，每個有 id、title、description、tags、imageUrl）。ProjectsPage 引入資料，用 .map() 渲染作品卡片，每張卡片包含一個 <Link to={`/projects/${project.id}`}> 連結到詳情頁。
實作 ProjectDetailPage（useParams）
在 ProjectDetailPage 中用 const { id } = useParams() 取得 ID，再從資料陣列中 .find() 對應的作品。若找不到（id 不存在），渲染錯誤訊息並附上返回作品列表的 Link。頁面顯示作品標題、詳細描述、技術標籤。
加入作品篩選功能（useSearchParams）
在 ProjectsPage 用 useSearchParams() 取得 ?tag=react 之類的篩選條件。根據 tag 篩選顯示的作品。建立技術標籤按鈕，點擊時用 setSearchParams({ tag: tagName }) 更新 URL。這樣篩選結果可以被分享（URL 包含篩選狀態）。
加分：實作保護路由
建立一個 src/context/AuthContext.jsx，用 useState 管理簡單的登入狀態（不需要真實 API，用按鈕切換 isLoggedIn 即可）。建立 PrivateRoute.jsx，未登入時跳轉到 /login，並用 state 記住原本路徑。新增 AdminPage，設為保護路由。
收尾：Scroll 回頂部與 404 優化
路由切換後滾動位置不會自動回頂，需要建立 ScrollToTop 元件：在元件中用 useEffect 監聽 location.pathname 變化，每次切換路由時執行 window.scrollTo(0, 0)。把這個元件放在 RootLayout 中使用。404 頁面加上有趣的設計和明確的返回指引。
