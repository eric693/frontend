02 — Destructuring

解構賦值

解構賦值讓你能從陣列或物件中「拆解」出值，賦予有意義的變數名稱。 它幾乎出現在所有現代 JS 程式碼中——函式參數、API 回應處理、React props 無不依賴它。

AD
陣列解構（Array Destructuring）
依照位置解構，名稱自訂

陣列解構依照索引位置取值。你可以自由命名變數、跳過不需要的元素、 設定預設值，以及用 rest 語法收集剩餘元素。

JavaScript — 陣列解構 
const scores = [98, 87, 73, 65, 50]

// 基本解構
const [first, second, third] = scores
// first=98, second=87, third=73

// 跳過元素（用逗號佔位）
const [top, , , fourth] = scores
// top=98, fourth=65

// 預設值（元素不存在時使用）
const [a, b, c, d, e, f = 0] = scores
// f=0 （索引5不存在）

// Rest：收集剩餘元素
const [best, ...rest] = scores
// best=98, rest=[87, 73, 65, 50]

// 交換兩個變數（無需暫存）
let x = 1, y = 2
;[x, y] = [y, x]
// x=2, y=1 ✓

// 搭配函式回傳值
function getRange() { return [0, 100] }
const [min, max] = getRange()
// min=0, max=100
OD
物件解構（Object Destructuring）
依照屬性名稱解構，與順序無關

物件解構依照屬性名稱取值（與位置無關），是最常用的解構形式。 React 元件幾乎每個函式參數都用物件解構來取 props。

JavaScript — 物件解構（含進階用法） 
const user = {
  name: "Eric",
  age: 25,
  role: "admin",
  address: { city: "Taipei", zip: "100" }
}

// 基本解構
const { name, age } = user
// name="Eric", age=25

// 重新命名（原名: 新名）
const { name: userName, role: userRole } = user
// userName="Eric", userRole="admin"

// 預設值（屬性不存在時使用）
const { name: n, score = 100 } = user
// score=100 （user 沒有 score 屬性）

// 巢狀解構
const { address: { city, zip } } = user
// city="Taipei", zip="100"

// Rest：收集剩餘屬性
const { name: uName, ...others } = user
// others = { age: 25, role: "admin", address: {...} }

// 函式參數解構（React props 常見寫法）
function UserCard({ name, age, role = "user" }) {
  return `${name} (${age}) — ${role}`
}
UserCard(user) // "Eric (25) — admin"
⚠ 巢狀解構的陷阱 解構巢狀屬性時，若中間層不存在會拋出 TypeError。 例如 const { a: { b } } = {} 會報錯，因為 a 是 undefined。 可搭配 ?. 或預設值保護：const { a: { b } = {} } = obj。
03 — Spread & Rest

展開與其餘運算子

... 是 ES6 中最靈活的語法之一，同一個符號在不同情境有截然不同的語意： 在「製造」資料時是 Spread（展開），在「收集」資料時是 Rest（其餘）。

Spread — 展開（建立新值）
// 展開陣列
const a = [1, 2, 3]
const b = [4, 5]
const merged = [...a, ...b]
// [1, 2, 3, 4, 5]

// 複製陣列（淺拷貝）
const copy = [...a]

// 展開物件
const base = { x: 1, y: 2 }
const updated = { ...base, y: 99, z: 3 }
// { x: 1, y: 99, z: 3 }

// 傳入函式
const nums = [3, 1, 4, 1, 5]
Math.max(...nums) // 5
Rest — 其餘（收集剩餘值）
// 函式參數 rest
function sum(first, ...others) {
  return first + others.reduce((a,b)=>a+b, 0)
}
sum(1,2,3,4) // 10

// 陣列解構 rest
const [head, ...tail] = [1,2,3,4]
// head=1, tail=[2,3,4]

// 物件解構 rest
const { id, ...data } = record
// 分離 id，其餘放進 data

// rest 永遠在最後一個
function log(level, ...msgs) {
  msgs.forEach(m => console.log(level, m))
}
JavaScript — 實際應用場景 
// ── 1. 不可變更新（React state 常見模式）──
const state = { user: "Eric", count: 0, theme: "dark" }

// 更新單一欄位，不改變原物件
const newState = { ...state, count: state.count + 1 }

// ── 2. 陣列操作（不修改原陣列）──
const todos = [{ id: 1, text: "Buy milk" }]

// 新增
const added = [...todos, { id: 2, text: "Read book" }]

// 刪除（過濾掉 id=1）
const removed = todos.filter(t => t.id !== 1)

// 更新（map 替換特定項目）
const updated = todos.map(t =>
  t.id === 1 ? { ...t, done: true } : t
)

// ── 3. 合併多個物件 ──
const defaults = { timeout: 5000, retries: 3, cache: true }
const userOpts = { timeout: 10000 }
const config = { ...defaults, ...userOpts }
// { timeout: 10000, retries: 3, cache: true }
// 後面的屬性會覆蓋前面
04 — Modern Operators

新式運算子

ES2020 之後陸續引入多個運算子，大幅簡化了空值檢查與條件賦值的寫法。 掌握這些運算子能讓你的程式碼更簡潔、意圖更清晰。

Optional Chaining
obj?.prop?.nested
安全存取深層屬性，任何一層為 null/undefined 時短路並回傳 undefined，不拋出 TypeError
Nullish Coalescing
value ?? fallback
只有左側為 null 或 undefined 時才取右側值。不同於 ||，0、""、false 不會被替換
Logical Assignment
a ??= b / ||= / &&=
結合邏輯運算子與賦值，只有在滿足條件時才賦值，避免不必要的覆蓋
JavaScript — Optional Chaining (?.) 完整用法 
const user = {
  profile: {
    avatar: "eric.jpg",
    social: { twitter: "@eric" }
  }
}

// 存取深層屬性
user?.profile?.social?.twitter  // "@eric"
user?.settings?.theme           // undefined（不拋錯）

// 呼叫可能不存在的方法
user?.profile?.getFullName?.()  // undefined（方法不存在）

// 存取動態屬性
const key = "twitter"
user?.profile?.social?.[key]    // "@eric"

// 陣列元素存取
const items = null
items?.[0]?.name  // undefined（不拋錯）

// 搭配解構
const { profile: { social: { twitter } = {} } = {} } = user
JavaScript — ?? vs || 的關鍵差異 
// || 遇到任何 falsy 值都觸發
const count1 = 0 || 10      // 10  ← 0 是 falsy！
const label1 = "" || "預設"  // "預設" ← 空字串是 falsy！
const show1  = false || true  // true ← false 是 falsy！

// ?? 只有 null / undefined 才觸發
const count2 = 0 ?? 10      // 0   ✓ 保留 0
const label2 = "" ?? "預設"  // ""  ✓ 保留空字串
const show2  = false ?? true  // false ✓ 保留 false
const name   = null ?? "匿名"  // "匿名" ← null 觸發

// Logical Assignment
let config = {}
config.timeout ??= 5000   // 若 timeout 是 null/undefined，設為 5000
config.name ||= "default" // 若 name 是 falsy，設為 "default"
config.debug &&= false   // 若 debug 是 truthy，設為 false
✅ 使用原則 當你想「有值才用那個值，沒值才用預設」，若值可能是 0、""、false 且你想保留， 請用 ??；若這些 falsy 值也應被替換，再用 ||。 大多數情況下 ?? 更安全、意圖更精確。
05 — ES6 Classes

Class 語法

ES6 Class 是 JavaScript 原型鏈繼承的語法糖，讓物件導向的寫法更直觀。 理解它的本質是原型，才能避免常見的誤解與陷阱。

JavaScript — Class 完整語法 
class Animal {
  // 私有欄位（ES2022，需要 # 前綴）
  #energy = 100

  // 靜態屬性
  static count = 0

  // 建構子
  constructor(name, species) {
    this.name = name
    this.species = species
    Animal.count++
  }

  // 實例方法
  speak() {
    return `${this.name} makes a sound`
  }

  // Getter / Setter
  get status() {
    return this.#energy > 50 ? "active" : "tired"
  }
  set energy(val) {
    this.#energy = Math.max(0, Math.min(100, val))
  }

  // 靜態方法
  static getCount() { return Animal.count }
}

// ── 繼承（extends + super）──
class Dog extends Animal {
  constructor(name, breed) {
    super(name, "Canis lupus")  // 必須先呼叫 super()
    this.breed = breed
  }

  // 覆寫父類別方法
  speak() {
    return `${this.name} barks!`
  }

  // 呼叫父類別方法
  introduce() {
    return `${super.speak()} — I'm a ${this.breed}`
  }
}

const dog = new Dog("Rex", "Labrador")
dog.speak()       // "Rex barks!"
dog.status        // "active" (getter)
dog.energy = 150  // setter 限制到 100
dog instanceof Animal  // true
Animal.getCount()       // 1
💡 Class 的本質 ES6 Class 只是原型繼承的語法糖。typeof Dog 是 "function"， 方法存在 Dog.prototype 上。理解這一點讓你能解釋 instanceof 的原理， 以及為何需要將 this 綁定到事件處理器。
JavaScript — 實戰：用 Class 封裝 API 請求 
class ApiClient {
  #baseUrl
  #headers

  constructor(baseUrl, token) {
    this.#baseUrl = baseUrl
    this.#headers = {
      "Content-Type": "application/json",
      "Authorization": `Bearer ${token}`
    }
  }

  async #request(method, path, body) {
    const res = await fetch(`${this.#baseUrl}${path}`, {
      method, headers: this.#headers,
      body: body ? JSON.stringify(body) : undefined
    })
    if (!res.ok) throw new Error(`HTTP ${res.status}`)
    return res.json()
  }

  get(path)        { return this.#request("GET",    path) }
  post(path, body) { return this.#request("POST",   path, body) }
  put(path, body)  { return this.#request("PUT",    path, body) }
  delete(path)     { return this.#request("DELETE", path) }
}

const api = new ApiClient("https://api.example.com", token)
const users = await api.get("/users")
const post  = await api.post("/posts", { title: "Hello" })
06 — Map & Set

Map 與 Set

ES6 引入了 Map 和 Set 兩種新的集合型別。 它們在特定場景下比普通物件和陣列更高效、語意更清晰。

比較Object / ArrayMap / Set
鍵的型別 只能是字串或 Symbol 任意型別（物件、函式、數字…）
順序保證 物件不保證（整數鍵除外） 保證插入順序
取得大小 Object.keys(o).length map.size（O(1)）
迭代 需要 Object.entries 轉換 直接 for...of 迭代
唯一值集合 需自行過濾 Set 天生保證唯一
Map — 任意鍵值對
const map = new Map()

// 任意型別都可以當 key
const objKey = { id: 1 }
map.set(objKey, "user data")
map.set(42, "number key")
map.set(true, "boolean key")

map.get(objKey)   // "user data"
map.has(42)       // true
map.size         // 3

// 用陣列初始化
const cache = new Map([
  ["user:1", { name: "Eric" }],
  ["user:2", { name: "Alex" }],
])

// 迭代
for (const [key, val] of cache) {
  console.log(key, val)
}
Set — 唯一值集合
const set = new Set([1, 2, 3, 2, 1])
// Set { 1, 2, 3 } — 自動去重

set.add(4)
set.has(2)     // true
set.delete(1)
set.size       // 3

// 陣列去重（最常用）
const arr = [1,2,2,3,3,3]
const unique = [...new Set(arr)]
// [1, 2, 3]

// 字串去重
const tags = new Set(["js", "ts", "js"])
// Set { "js", "ts" }

// 集合運算
const a = new Set([1,2,3])
const b = new Set([2,3,4])
// 交集
const inter = [...a].filter(x => b.has(x))
// 聯集
const union = new Set([...a, ...b])
07 — ES Modules

模組化：import / export

ES Module 是 JavaScript 官方的模組系統（ES2015），讓你把程式碼拆分成多個檔案， 每個檔案有自己的作用域，透過 export 公開介面、import 引用依賴。

utils/api.js
→ export
fetchWeather, fetchMovies, ApiClient
utils/storage.js
→ export
saveToStorage, loadFromStorage, clearStorage
utils/format.js
→ export
formatDate, formatTemp, formatWind
main.js
← import
引用以上所有工具函式，組合成應用程式
具名匯出（Named Export）
// utils/math.js
export const PI = 3.14159

export function add(a, b) {
  return a + b
}

export function multiply(a, b) {
  return a * b
}

// 或集中匯出
export { PI, add, multiply }

// 引用（名稱必須對應）
import { add, multiply } from "./utils/math.js"
import { add as sum } from "./utils/math.js"
import * as Math2 from "./utils/math.js"
預設匯出（Default Export）
// utils/api.js
class ApiClient {
  constructor(baseUrl) {
    this.baseUrl = baseUrl
  }
  async get(path) { /* ... */ }
}

// 每個檔案只能有一個 default
export default ApiClient

// 引用（名稱可自訂）
import ApiClient from "./utils/api.js"
import Client    from "./utils/api.js"

// 混用
import ApiClient, { BASE_URL } from "./utils/api.js"
JavaScript — 動態 import() 與進階用法 
// ── 動態 import（按需載入）──
async function loadChart() {
  // 只有在需要時才載入，減少初始 bundle 大小
  const { Chart } = await import("./chart.js")
  new Chart("#canvas", data)
}

// ── Re-export（模組聚合）──
// utils/index.js — 統一出口，外部只需引一個檔案
export { add, multiply } from "./math.js"
export { default as ApiClient } from "./api.js"
export * from "./format.js"

// 外部引用
import { add, ApiClient, formatDate } from "./utils/index.js"

// ── HTML 中使用 ES Module ──
// <script type="module" src="main.js"></script>
// type="module" 讓瀏覽器以模組模式解析
// 自動啟用嚴格模式，頂層作用域不是全域
⚠ 模組的注意事項 ES Module 使用 靜態分析，import 必須在檔案頂層，不能在條件判斷或迴圈內。 模組在瀏覽器中有 CORS 限制，本地直接開啟 HTML 檔案（file://）會報錯，需要透過 HTTP server 提供服務。
08 — Quiz

隨堂測驗

用以下題目確認你對 ES6+ 語法的掌握程度。

QUESTION 01 / 04
以下程式碼執行後，x 和 y 的值是多少？
const [x, , y = 10] = [1, 2]
Ax = 1, y = 2
Bx = 1, y = 10
Cx = 1, y = undefined
D語法錯誤
正確！[x, , y = 10] 中第二個元素（2）被跳過，y 對應索引2，但陣列只有兩個元素，故使用預設值 10。
QUESTION 02 / 04
0 ?? "default" 與 0 || "default" 的結果分別是什麼？
A兩者都回傳 "default"
B兩者都回傳 0
C?? 回傳 0，|| 回傳 "default"
D?? 回傳 "default"，|| 回傳 0
正確！?? 只有 null/undefined 才觸發，0 不是 null/undefined，所以回傳 0。|| 遇到所有 falsy 值都觸發，0 是 falsy，所以回傳 "default"。
QUESTION 03 / 04
以下哪一行會正確地將陣列 [1,2,2,3,3,3] 去除重複值，得到 [1,2,3]？
Anew Set([1,2,2,3,3,3])
B[...new Set([1,2,2,3,3,3])]
CArray.from([1,2,2,3,3,3])
D[1,2,2,3,3,3].unique()
正確！new Set() 建立的是 Set 物件，需要用 Spread 或 Array.from() 轉回陣列。Array.from([1,2,2,3,3,3]) 只是淺拷貝，不去重。陣列沒有 .unique() 方法。
QUESTION 04 / 04
關於 ES Module，下列哪個敘述正確？
A每個檔案可以有多個 export default
Bimport 可以寫在函式內部或條件判斷中
C具名匯出（named export）引用時名稱必須與匯出時相符，或用 as 重新命名
DES Module 不支援在瀏覽器中使用，需要 bundler 轉換
正確！具名匯出的引用必須使用相同名稱，或透過 import { foo as bar } 重新命名。每個檔案只能有一個 default export，import 必須在頂層，現代瀏覽器原生支援 ES Module（需 type="module"）。
